## Installing Kafka

Follow the official quick start guide: [kafka.apache.org/quickstart](https://kafka.apache.org/quickstart)

### Step 1 — Download & Extract

Download Kafka from the quick start page and unzip. After extracting, the folder structure looks like:

```
kafka/
  ├── bin/                  ← shell scripts for all Kafka operations
  │     └── windows/        ← .bat equivalents for Windows
  ├── config/
  │     ├── kraft/          ← KRaft config files (use these, not ZooKeeper)
  │     │     ├── server.properties    ← main config file
  │     │     ├── broker.properties
  │     │     └── controller.properties
  │     └── server.properties  ← ZooKeeper version (ignore this)
  └── ...
```

> **macOS/Linux**: use `.sh` scripts in `bin/`  
> **Windows**: use `.bat` scripts in `bin/windows/`

---

### Step 2 — Configure (`server.properties`)

Open `config/kraft/server.properties` — key settings for local single broker setup:

```properties
# Role of this broker (broker + controller in KRaft mode)
process.roles=broker,controller

# Unique node ID — change this if running multiple brokers
node.id=1

# Kafka runs on this port
listeners=PLAINTEXT://:9092

# Default number of partitions per topic
num.partitions=1

# Log retention — messages kept for 168 hours (7 days) by default
log.retention.hours=168
```

> **Disable auto-topic-creation in production**: set `auto.create.topics.enable=false` on the broker. Otherwise any consumer/producer typo creates a topic with default partitioning and no replication, which becomes a silent operational liability. Topics should be created intentionally via CLI/Terraform/managed-service tooling.

#### Advanced Config (Multi-broker / Docker / Cloud)

These are already set correctly in the default `server.properties` for single broker local setup. Configure these when running multiple brokers or deploying to Docker/cloud:

```properties
# Required for controller election — format: nodeId@host:controllerPort
controller.quorum.voters=1@localhost:9093

# Separate listener for controller communication
listeners=PLAINTEXT://:9092,CONTROLLER://:9093
listener.security.protocol.map=PLAINTEXT:PLAINTEXT,CONTROLLER:PLAINTEXT
controller.listener.names=CONTROLLER

# What clients actually connect to — critical in Docker/cloud
# (may differ from listeners when behind NAT or in containers)
advertised.listeners=PLAINTEXT://localhost:9092

# Where Kafka stores data — must be unique per broker to avoid clashes
log.dirs=/tmp/kafka-logs
```

---

### Step 3 — Start Kafka (KRaft mode)

```bash
# 1. Generate a cluster ID
KAFKA_CLUSTER_ID="$(bin/kafka-storage.sh random-uuid)"
```

> **Important:** This cluster ID must remain consistent across restarts — do not regenerate it for the same data directory. Only generate once per cluster setup.

```bash
# 2. Format the log directories
bin/kafka-storage.sh format -t $KAFKA_CLUSTER_ID -c config/kraft/server.properties

# 3. Start Kafka server
bin/kafka-server-start.sh config/kraft/server.properties
```

> Kafka runs on **port 9092** by default.  
> If you get an "Invalid Cluster ID" error on step 2, your log directory was already formatted with a different cluster ID — reuse the existing one or delete the log directory and reformat.

---

### Step 4 — Create a Topic

```bash
# Create a topic with explicit partition and replication factor
bin/kafka-topics.sh --create \
  --topic my-topic \
  --partitions 1 \
  --replication-factor 1 \
  --bootstrap-server localhost:9092

# Describe the topic (partitions, replication factor, etc.)
bin/kafka-topics.sh --describe \
  --topic my-topic \
  --bootstrap-server localhost:9092
```

---

### Step 5 — Produce & Consume Events

Open two separate terminals — keep Kafka running in the background:

**Terminal 1 — Producer**

```bash
bin/kafka-console-producer.sh \
  --topic my-topic \
  --bootstrap-server localhost:9092
>hello
>world
>this
>is
>kafka
```

**Terminal 2 — Consumer**

```bash
# --from-beginning reads all existing messages
# Without it, consumer only reads new messages arriving after it connects
bin/kafka-console-consumer.sh \
  --topic my-topic \
  --group test-group \
  --from-beginning \
  --bootstrap-server localhost:9092
hello
world
this
is
kafka
```

> Messages typed in Terminal 1 appear in Terminal 2 in real time.

## Kafbat UI

A versatile, fast, and lightweight web UI for managing Apache Kafka clusters — visualize topics, partitions, messages, consumers, and brokers.

Installation guide: [ui.docs.kafbat.io/development/building/without-docker](https://ui.docs.kafbat.io/development/building/without-docker)

> Requires **Java 17+** to run.

### Setup

**Step 1**: Download the latest `kafbat-ui.jar` from the installation page.

**Step 2**: Create `application-local.yml` in the same folder:

```yaml
logging:
  level:
    root: INFO

spring:
  jmx:
    enabled: true

kafka:
  clusters:
    - name: local
      bootstrapServers: localhost:9092
```

**Step 3**: Run Kafbat UI:

```bash
java -Dspring.config.additional-location=application-local.yml \
     -jar kafbat-ui.jar
```

Kafbat UI runs on **port 8080** → open `http://localhost:8080`

### What You Can See in Kafbat UI

- **Topics**: list of all topics, partition count, replication factor
- **Messages**: browse messages with offset, partition, timestamp, key, value, headers
- **Consumers**: active consumer groups, subscribed topics, consumer lag
- **Brokers**: broker count, disk usage, cluster info

## Running Multiple Brokers Locally

To run multiple brokers on the same machine:

1. Create separate `server.properties` files for each broker with different `node.id`, `listeners` port, and `log.dirs` values — brokers will clash if they share the same port or data directory
2. Start each broker in a separate terminal

> Follow this guide **with Docker**: [kafka-stack-docker-compose — conduktor](https://github.com/conduktor/kafka-stack-docker-compose/blob/master/zk-single-kafka-multiple.yml)  
> Follow this guide **without Docker**: [Kafka KRaft multi-broker setup — Medium](https://medium.com/@aparna1002v/kafka-kraft-setup-multiple-kafka-brokers-in-localhost-9a5ec2cc257e)
