Local single-broker setup in KRaft mode, following the official [quickstart](https://kafka.apache.org/quickstart). On Kafka 4.0+ there is no ZooKeeper: the top-level `config/server.properties` is the KRaft config, and the old `config/kraft/` directory and `config/zookeeper.properties` are gone.

```
kafka/
  bin/                    shell scripts (.bat equivalents under bin/windows/)
  config/
    server.properties     KRaft config (broker + controller); edit this
```

> **Note**: use the `.sh` scripts in `bin/` on macOS/Linux and the `.bat` scripts in `bin/windows/` on Windows.

---

## Configure

Key settings in `config/server.properties` for a local single broker (combined broker + controller):

```properties
process.roles=broker,controller          # this node is both broker and controller
node.id=1                                # unique per broker
controller.quorum.voters=1@localhost:9093
listeners=PLAINTEXT://:9092,CONTROLLER://:9093
controller.listener.names=CONTROLLER
log.dirs=/tmp/kraft-combined-logs        # unique per broker
num.partitions=1
log.retention.hours=168                  # 7 days
```

For Docker/cloud, also set `advertised.listeners` to what clients actually connect to (it differs from `listeners` behind NAT or in containers).

> **Note**: disable auto-topic-creation in production (`auto.create.topics.enable=false`). Otherwise a producer/consumer typo silently creates a topic with default partitioning and no replication. Topics should be created intentionally via CLI/IaC/managed-service tooling.

## Start Kafka

Generate a cluster ID once, format the log directory with it, then start the broker:

```bash
KAFKA_CLUSTER_ID="$(bin/kafka-storage.sh random-uuid)"
bin/kafka-storage.sh format -t "$KAFKA_CLUSTER_ID" -c config/server.properties
bin/kafka-server-start.sh config/server.properties
```

> **Note**: keep the cluster ID consistent across restarts for the same data directory; generate it only once. An "Invalid Cluster ID" error on format means the directory was already formatted with a different ID, so reuse the existing one or delete `log.dirs` and reformat. Kafka listens on port 9092 by default.

## Create a Topic and Test

```bash
bin/kafka-topics.sh --create --topic my-topic \
  --partitions 1 --replication-factor 1 --bootstrap-server localhost:9092

bin/kafka-topics.sh --describe --topic my-topic --bootstrap-server localhost:9092
```

Produce and consume in two terminals; `--from-beginning` replays existing messages instead of only new ones:

```bash
# producer
bin/kafka-console-producer.sh --topic my-topic --bootstrap-server localhost:9092

# consumer
bin/kafka-console-consumer.sh --topic my-topic --group test-group \
  --from-beginning --bootstrap-server localhost:9092
```

## Kafbat UI

A lightweight web UI for inspecting clusters: topics (partition count, RF), messages (offset, partition, key, value, headers), consumer groups and lag, and broker info. It requires Java 17+. Download `kafbat-ui.jar`, add a config file, and run it on port 8080:

```yaml
# application-local.yml
kafka:
  clusters:
    - name: local
      bootstrapServers: localhost:9092
```

```bash
java -Dspring.config.additional-location=application-local.yml -jar kafbat-ui.jar
# open http://localhost:8080
```

## Running Multiple Brokers Locally

Give each broker its own `server.properties` with a distinct `node.id`, `listeners` port, and `log.dirs` (they clash on a shared port or data directory), then start each in its own terminal. For a faster path, use a Docker Compose stack such as [conduktor/kafka-stack-docker-compose](https://github.com/conduktor/kafka-stack-docker-compose).

## Quick Revision Cheat Sheet

| Step                | Command / setting                                              |
| ------------------- | ------------------------------------------------------------- |
| Config file         | `config/server.properties` (KRaft; no ZooKeeper in 4.0+)      |
| Roles               | `process.roles=broker,controller` for a combined node         |
| Format storage      | `kafka-storage.sh format -t <id> -c config/server.properties` |
| Start broker        | `kafka-server-start.sh config/server.properties`              |
| Default port        | 9092                                                          |
| Create topic        | `kafka-topics.sh --create --partitions N --replication-factor R` |
| Replay from start   | `--from-beginning` on the console consumer                    |
| Production safety    | `auto.create.topics.enable=false`                            |
