## Spring Boot Test Dependency

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

This single dependency includes everything you need for testing:

|Library|Purpose|
|---|---|
|**JUnit 5**|Testing framework — structure, run, and report tests|
|**AssertJ**|Fluent assertion library|
|**Mockito**|Mocking framework|
|**Spring Test**|Spring-specific test utilities|
|**JSONassert**|JSON comparison assertions|
|**JsonPath**|JSON path expressions for response testing|

> You don't need to add JUnit, AssertJ, or Mockito separately — they all come with `spring-boot-starter-test`.

## JUnit 5

The most widely used testing framework in Java. Provides annotations to structure your tests and a basic set of assertions to verify results.

**Test lifecycle:**

```
@BeforeAll   → runs once before all tests (static method)
       │
       ▼
┌─────────────────────────────┐
│  @BeforeEach → setUp()      │
│         │                   │
│         ▼                   │
│    @Test → test method      │  ← repeats for each test
│         │                   │
│         ▼                   │
│  @AfterEach → tearDown()    │
└─────────────────────────────┘
       │
       ▼
@AfterAll    → runs once after all tests (static method)
```

**Annotations reference:**

|Annotation|Purpose|Runs|
|---|---|---|
|`@Test`|Marks a method as a test|Each test|
|`@BeforeEach`|Setup before each test (e.g., reset data)|Before every `@Test`|
|`@AfterEach`|Cleanup after each test|After every `@Test`|
|`@BeforeAll`|One-time setup (must be `static`)|Once before all tests|
|`@AfterAll`|One-time cleanup (must be `static`)|Once after all tests|
|`@DisplayName`|Custom name for test reports and IDE|—|
|`@Disabled`|Skip this test|—|

> `@BeforeAll` and `@AfterAll` must be `static` because they run before any instance of the test class is created.

**Example:**

```java
class UserServiceTest {

    @BeforeAll
    static void setUpOnce() {
        // runs once — e.g., start test database
    }

    @AfterAll
    static void tearDownOnce() {
        // runs once — e.g., stop test database
    }

    @BeforeEach
    void setUp() {
        // runs before each test — e.g., reset test data
    }

    @AfterEach
    void tearDown() {
        // runs after each test — e.g., clean up
    }

    @Test
    @DisplayName("Should find user by email")
    void shouldFindUserByEmail() {
        // test logic + assertions
    }

    @Test
    @Disabled("TODO: implement after auth is done")
    void shouldUpdateUserRoles() {
        // skipped
    }
}
```

**Parameterized tests**: run the same test logic with multiple inputs without copy-pasting:

```java
@ParameterizedTest
@ValueSource(strings = { "user@example.com", "admin@example.com", "alice@x.io" })
void emailValidator_acceptsValid(String email) {
    assertThat(EmailValidator.isValid(email)).isTrue();
}

@ParameterizedTest
@CsvSource({
    "100, 10, 110",
    "0,   0,  0",
    "-5,  5,  0"
})
void calculator_addsCorrectly(int a, int b, int expected) {
    assertThat(Calculator.add(a, b)).isEqualTo(expected);
}

@ParameterizedTest
@MethodSource("invalidPasswords")
void passwordValidator_rejectsBad(String password, String reason) {
    assertThat(PasswordValidator.isValid(password))
        .as("rejecting because: %s", reason)
        .isFalse();
}

static Stream<Arguments> invalidPasswords() {
    return Stream.of(
        arguments("short",      "too short"),
        arguments("nouppercase","no uppercase"),
        arguments("NoDigits!",  "no digits")
    );
}

@ParameterizedTest
@EnumSource(Role.class)
void role_isAlwaysPrefixedWithRoleUnderscore(Role role) {
    assertThat(role.getAuthority()).startsWith("ROLE_");
}
```

| Source | Use when |
|---|---|
| `@ValueSource` | Single argument from a constant list (strings, ints, longs, classes) |
| `@CsvSource` | Multiple primitive arguments per row, inline |
| `@CsvFileSource` | Same but from a CSV file in resources |
| `@MethodSource` | Complex objects, computed inputs, or anything needing logic |
| `@EnumSource` | Test all values of an enum |

> **Test report shows each input as its own line**: when one row fails, you see exactly which input broke without scanning a single bloated test. Replaces 5–10 copy-pasted `@Test` methods.

## AssertJ

JUnit's built-in assertions work, but have rough edges:

```java
assertEquals("John", result.getName());   // expected? actual? easy to flip
assertNotNull(result);
assertTrue(list.contains("apple"));       // failure: "expected true but was false" — what was the list?
assertThrows(NotFoundException.class, () -> service.getById(999L));
```

Problems: parameter order is ambiguous, failure messages are generic, you can't chain checks, and there are no type-specific helpers (no `startsWith`, `hasSize`, `containsExactly`).

**AssertJ** fixes all of this with a single entry point — `assertThat()` — that detects the type you pass in and exposes type-specific methods:

```java
import static org.assertj.core.api.Assertions.assertThat;
import static org.assertj.core.api.Assertions.assertThatThrownBy;

// No parameter order confusion — always assertThat(actual)
assertThat(result.getName()).isEqualTo("John");

// Clear failure messages — shows what the list actually had
assertThat(list).contains("apple");

// Fluent chaining
assertThat(result)
    .isNotNull()
    .extracting("name", "email")
    .containsExactly("John", "john@example.com");

// Type-specific methods
assertThat("hello").startsWith("he").contains("ell");
assertThat(list).hasSize(2).contains("apple");
```

> **Use AssertJ for all assertions.** JUnit assertions exist but AssertJ is the standard in modern Spring Boot projects. Both are included in `spring-boot-starter-test`.

## AssertJ by Type

**Numbers:**

```java
assertThat(5)
    .isEqualTo(5)
    .isNotEqualTo(10)
    .isGreaterThan(4)
    .isLessThanOrEqualTo(5)
    .isBetween(1, 10);
```

**Strings:**

```java
assertThat("hello")
    .startsWith("he")
    .endsWith("lo")
    .contains("ell")
    .hasSize(5)
    .isNotBlank();
```

**Booleans:**

```java
assertThat(true).isTrue();
assertThat(false).isFalse();
```

**Objects:**

```java
assertThat(result)
    .isNotNull()
    .isInstanceOf(UserResponse.class);

assertThat(result.getEmail())
    .isEqualTo("john@example.com");
```

**Lists / arrays:**

```java
assertThat(List.of("apple", "banana"))
    .contains("apple")
    .doesNotContain("orange")
    .hasSize(2)
    .isNotEmpty();
```

**Exceptions:**

```java
assertThatThrownBy(() -> {
    userService.getUserById(999L);
})
    .isInstanceOf(ResourceNotFoundException.class)
    .hasMessage("User not found with id: 999")
    .hasMessageContaining("not found");
```

## Q&A

1. **Why must `@BeforeAll` and `@AfterAll` be `static`?**
   They run before any instance of the test class is created, so they can't rely on instance state. JUnit 5 does allow non-static with `@TestInstance(Lifecycle.PER_CLASS)`, but the default (`PER_METHOD`) requires static.

2. **What's the difference between `@BeforeEach` and `@BeforeAll`?**
   `@BeforeEach` runs before every `@Test` method — use it for per-test setup (resetting data, fresh mocks). `@BeforeAll` runs once for the whole class — use it for expensive one-time setup like starting a container.

3. **Why prefer AssertJ over JUnit's built-in assertions?**
   AssertJ is fluent (`assertThat(x).isEqualTo(y)` reads naturally), gives descriptive failure messages with actual values, supports chaining, and exposes type-specific methods (`.startsWith`, `.hasSize`, `.containsExactly`). JUnit assertions are terse but error-prone (expected/actual order confusion).

4. **How do you assert that code throws an exception?**
   `assertThatThrownBy(() -> service.getById(999L)).isInstanceOf(NotFoundException.class).hasMessageContaining("not found")`. JUnit equivalent is `assertThrows(NotFoundException.class, () -> ...)`.

5. **What does `spring-boot-starter-test` include?**
   JUnit 5, AssertJ, Mockito, Spring Test utilities, JSONassert, and JsonPath — everything needed for unit and integration testing in one dependency.

## Quick Reference

|Concept|Remember|
|---|---|
| `@Test` | Marks a method as a test case |
| `@BeforeEach` / `@AfterEach` | Runs before/after every test — per-test setup/cleanup |
| `@BeforeAll` / `@AfterAll` | Runs once per class — must be `static` |
| `@DisplayName` | Custom human-readable name in reports |
| `assertThat(actual)` | AssertJ entry point — fluent, type-aware |
| `assertThatThrownBy(() -> ...)` | Assert an exception is thrown |
| `spring-boot-starter-test` | JUnit + AssertJ + Mockito + Spring Test, all in one |
