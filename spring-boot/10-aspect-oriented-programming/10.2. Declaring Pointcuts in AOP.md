A **pointcut** is an expression that tells Spring which methods your advice should apply to. You write a pattern, Spring matches it against your methods.

```
Pointcut: "All methods in the service package"
       │
       ▼
Spring finds matching methods
       │
       ▼
Applies your advice (logging, security, etc.) to those methods
```

---

## Pointcut Types

### `execution` — Target Specific Methods (Most Common)

Matches method executions based on return type, class, method name, and parameters.

```
execution(return-type package.class.method(params))
```

```java
// All methods in service package
@Before("execution(* com.app.service.*.*(..))")

// Breakdown:
// *                    → any return type
// com.app.service      → package
// *                    → any class
// *                    → any method
// (..)                 → any parameters
```

**More examples:**

```java
// Only methods that return String
@Before("execution(String com.app.service.*.*(..))")

// Only getEmployee method in EmployeeService
@Before("execution(* com.app.service.EmployeeService.getEmployee(..))")

// Only methods starting with "get"
@Before("execution(* com.app.service.*.get*(..))")

// Methods with exactly one Long parameter
@Before("execution(* com.app.service.*.*(Long))")

// Methods with no parameters
@Before("execution(* com.app.service.*.*())")

// All methods in service package and sub-packages
@Before("execution(* com.app.service..*.*(..))")
//                                    ^^ double dot = include sub-packages
```

**Parameter patterns:**

|Pattern|Matches|
|---|---|
|`()`|No parameters|
|`(Long)`|One Long parameter|
|`(Long, String)`|Exactly Long then String|
|`(..)`|Any number and type of parameters|
|`(*, ..)`|At least one parameter of any type|

### `within` — Target by Class or Package

Matches **all methods** within a class or package. Simpler than `execution` when you don't care about specific methods.

```java
// All methods in EmployeeService
@Before("within(com.app.service.EmployeeServiceImpl)")

// All methods in service package
@Before("within(com.app.service.*)")

// All methods in service package and sub-packages
@Before("within(com.app.service..*)")
```

**`execution` vs `within`:**

|                        | execution        | within                  |
|------------------------|------------------|-------------------------|
| Target                 | Specific methods | Entire class or package |
| Filter by method name  | Yes              | No                      |
| Filter by return type  | Yes              | No                      |
| Filter by parameters   | Yes              | No                      |
| Simpler syntax         | No               | Yes                     |

> Use `execution` when you need to target specific methods. Use `within` when you want all methods in a class/package.

### `@annotation` — Target by Annotation

Matches methods that have a specific annotation. Works with built-in or **custom** annotations.

```java
// All methods annotated with @Transactional
@Before("@annotation(org.springframework.transaction.annotation.Transactional)")

// All methods annotated with a custom annotation
@Before("@annotation(com.app.annotation.LogExecutionTime)")
```

**Creating a custom annotation:**

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface LogExecutionTime {
}
```

**Using it:**

```java
@Service
public class EmployeeServiceImpl {

    @LogExecutionTime
    public EmployeeDto getEmployeeById(Long id) {
        // only this method gets the advice
    }

    public List<EmployeeDto> getAllEmployees() {
        // this method is NOT targeted
    }
}
```

> `@annotation` gives you the most control — you choose exactly which methods to target by placing an annotation on them.

## JoinPoint — Extracting Method Info

The `JoinPoint` parameter gives you access to information about the method being intercepted:

```java
@Before("execution(* com.app.service.*.*(..))")
public void logBefore(JoinPoint joinPoint) {

    // Method name
    String methodName = joinPoint.getSignature().getName();
    // → "getEmployeeById"

    // Full signature
    String fullSignature = joinPoint.getSignature().toString();
    // → "EmployeeDto com.app.service.EmployeeService.getEmployeeById(Long)"

    // Arguments passed to the method
    Object[] args = joinPoint.getArgs();
    // → [1L]

    // Class where the method belongs
    String className = joinPoint.getTarget().getClass().getSimpleName();
    // → "EmployeeServiceImpl"

    log.info("{}.{} called with args: {}",
        className, methodName, Arrays.toString(args));
}
```

```
Output: EmployeeServiceImpl.getEmployeeById called with args: [1]
```

## Reusable Pointcut Declaration

When multiple advice methods target the same methods, repeating the expression gets messy:

```java
// Without reusable pointcut — repeated expression
@Before("execution(* com.app.service.*.*(..))")
public void logBefore() { }

@After("execution(* com.app.service.*.*(..))")
public void logAfter() { }
```

Define it once with `@Pointcut`, reuse by name:

```java
@Aspect
@Component
public class LoggingAspect {

    // Define once
    @Pointcut("execution(* com.app.service.*.*(..))")
    public void serviceMethods() {}

    @Pointcut("execution(* com.app.controller.*.*(..))")
    public void controllerMethods() {}

    // Reuse by name
    @Before("serviceMethods()")
    public void logBeforeService(JoinPoint joinPoint) {
        log.info("Service: {}", joinPoint.getSignature().getName());
    }

    @After("serviceMethods()")
    public void logAfterService(JoinPoint joinPoint) {
        log.info("Service completed: {}", joinPoint.getSignature().getName());
    }
}
```

## Combining Pointcuts

```java
// Both service AND controller methods
@Before("serviceMethods() || controllerMethods()")

// Service methods but NOT get methods
@Before("serviceMethods() && !execution(* com.app.service.*.get*(..))")
```

|Operator|Meaning|
|---|---|
|`\|\|`|OR — matches either pointcut|
|`&&`|AND — matches both pointcuts|
|`!`|NOT — excludes matching methods|

## Q&A

1. **What is a pointcut?**
   An expression that selects which join points (method executions, in Spring AOP) an advice applies to. You write a pattern; Spring matches it against the methods in your beans.

2. **Difference between `execution` and `within`?**
   `execution` can filter by return type, method name, and parameters — precise targeting. `within` matches all methods in a given class or package — coarser but simpler. Use `execution` for specific methods, `within` for broad class/package targeting.

3. **How do you target methods with a specific annotation?**
   Use `@annotation(fully.qualified.Annotation)`. This is especially useful with **custom annotations**: place the annotation on methods you want advised, so targeting is opt-in rather than path-based.

4. **What does `JoinPoint` give you access to?**
   Method signature (`getSignature()`), arguments (`getArgs()`), and the target object (`getTarget()`). Used inside advice to log, inspect, or route based on runtime method info.

5. **Why declare a named `@Pointcut`?**
   To avoid repeating the same expression across multiple advice methods. You define it once as a method with `@Pointcut("...")` and reference it by name (`serviceMethods()`), keeping advice declarations clean.

6. **How do you combine pointcuts?**
   With `&&` (both must match), `||` (either matches), and `!` (negation) — e.g., `serviceMethods() && !execution(* *.get*(..))` targets service methods that aren't getters.

## Quick Reference

|Type|Use When|Example|
|---|---|---|
|`execution`|Target specific methods by name, return type, params|`execution(* com.app.service.*.get*(..))`|
|`within`|Target all methods in a class or package|`within(com.app.service.*)`|
|`@annotation`|Target methods with a specific annotation|`@annotation(com.app.annotation.LogExecutionTime)`|
|`&&` / `\|\|` / `!`|Combine / OR / negate pointcuts|`serviceMethods() && !execution(* *.get*(..))`|
