## Project Setup

### Step 1: Create Services from Spring Initializr

Go to [start.spring.io](https://start.spring.io/) and create two projects:

**Inventory Service**: Group: `com.example.ecommerce`
- Artifact: `inventory-service`
- Dependencies: Spring Web, Spring Data JPA, PostgreSQL Driver, Lombok

**Order Service**: Group: `com.example.ecommerce`
- Artifact: `order-service`
- Dependencies: Spring Web, Spring Data JPA, PostgreSQL Driver, Lombok

### Step 2: Organize into One Folder

```bash
mkdir ecommerce
# Download both zips into ecommerce/
cd ecommerce
unzip inventory-service.zip
unzip order-service.zip
rm *.zip
```

```
ecommerce/
  ├── inventory-service/
  │     └── src/main/java/com/example/ecommerce/inventory_service/
  └── order-service/
        └── src/main/java/com/example/ecommerce/order_service/
```

> Open the `ecommerce` folder in IntelliJ — it will detect both projects.

## Inventory Service — Entities and Structure

```
inventory-service/src/main/java/com/example/ecommerce/inventory_service/
  ├── entity/
  │     └── Product.java
  ├── repository/
  │     └── ProductRepository.java
  ├── service/
  │     └── ProductService.java
  ├── controller/
  │     └── ProductController.java
  └── dto/
        └── ProductDto.java
```

```java
@Entity
@Getter @Setter @NoArgsConstructor @AllArgsConstructor @Builder
public class Product {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String title;

    private String description;

    @Column(nullable = false)
    private BigDecimal price;

    @Column(nullable = false)
    private Integer stock;
}
```

```yaml
# inventory-service application.yml
server:
  port: 8082
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/inventory_db
    username: postgres
    password: postgres
  jpa:
    hibernate:
      ddl-auto: update
```

## Order Service — Entities and Structure

```
order-service/src/main/java/com/example/ecommerce/order_service/
  ├── entity/
  │     ├── Order.java
  │     ├── OrderItem.java
  │     └── OrderStatus.java
  ├── repository/
  │     └── OrderRepository.java
  ├── service/
  │     └── OrderService.java
  ├── controller/
  │     └── OrderController.java
  └── dto/
        └── OrderDto.java
```

```java
@Entity
@Getter @Setter @NoArgsConstructor @AllArgsConstructor @Builder
public class Order {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Enumerated(EnumType.STRING)
    private OrderStatus status;

    private BigDecimal totalAmount;

    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL)
    private List<OrderItem> items;
}
```

```java
public enum OrderStatus {
    PENDING, CONFIRMED, CANCELLED
}
```

```java
@Entity
@Getter @Setter @NoArgsConstructor @AllArgsConstructor @Builder
public class OrderItem {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long productId;  // references Product in Inventory Service

    private Integer quantity;

    private BigDecimal price;

    @ManyToOne
    @JoinColumn(name = "order_id")
    private Order order;
}
```

```yaml
# order-service application.yml
server:
  port: 8081
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/order_db
    username: postgres
    password: postgres
  jpa:
    hibernate:
      ddl-auto: update
```

> Each service has its own database — `inventory_db` and `order_db`. This is the **database per service** pattern in microservices.

> **Production warning:** `ddl-auto=update` is fine for local development only. In production, use `validate` plus a migration tool (Flyway / Liquibase) so schema changes are versioned and reviewed — never let JPA mutate the schema in prod.

## Key Points and What's Next

- **Separate databases**: services don't share tables. Order Service stores `productId` as a plain reference, not a foreign key to Inventory Service's `Product` table.
- **Separate ports**: Order on 8081, Inventory on 8082. Both run independently.
- **No direct communication yet**: Order Service can't call Inventory Service. Covered in [12.5 Service-to-Service Communication with OpenFeign](<12.5. Service-to-Service Communication with OpenFeign.md>).
- **No service discovery yet**: services don't know about each other. Covered in [12.3 Service Registration and Discovery with Eureka](<12.3. Service Registration and Discovery with Eureka.md>).

## Q&A

1. **Why does each microservice have its own database?** To enforce loose coupling — one service can evolve its schema without breaking others. Cross-service data access happens via APIs, not shared tables.
2. **How does Order Service reference a Product owned by Inventory Service?** By storing `productId` as a plain `Long` — not a JPA foreign key. The reference is logical; integrity is enforced by the Inventory Service API, not the database.
3. **Why separate ports per service locally?** Each service is an independent Spring Boot app with its own embedded server — they can't share a port. In production each runs in its own container/host.

## Quick Reference

| Concept | Remember |
|---|---|
| Database per service | Each service owns its DB — no shared tables, no cross-service joins |
| Cross-service references | Store the foreign id (`productId`) as a plain `Long`, not a JPA `@ManyToOne` |
| Local ports | Each service gets its own port (Order 8081, Inventory 8082) |
| `ddl-auto` | `update` for dev only; production uses `validate` + Flyway/Liquibase |
| Folder layout | Sibling project folders under one parent (`ecommerce/`) — IntelliJ detects both |
