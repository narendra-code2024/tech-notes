## Comparison

| Aspect | Spring Framework | Spring Boot |
|---|---|---|
| Definition | Core Java framework for enterprise apps | **Opinionated** framework built on Spring |
| Setup | **Manual** configuration | **Auto-configuration** |
| Configuration | **XML** / Java config / annotations | Mostly **annotations** & **properties** |
| Dependency management | Manual versions | **Starter dependencies** (version managed) |
| Server configuration | **External** server (Tomcat, etc.) | **Embedded** server (Tomcat/Jetty/Undertow) |
| Deployment | **WAR** → external server | **JAR** (standalone, self-contained) |
| Production-ready features | Manual setup | Built-in (**Actuator**, metrics, health checks) |
| Configuration files | `applicationContext.xml` | `application.properties` / `application.yml` |
| Microservices support | Possible but complex | **Ideal** & widely used |
| Testing support | Manual setup | Auto-configured (`@SpringBootTest`) |
| Use case | Large traditional **enterprise** apps | Modern apps, REST APIs, **microservices** |

> **Note:** Spring Boot startup can be *slower* than a minimal Spring Framework app due to auto-configuration and classpath scanning. The advantage is faster *development*, not necessarily faster runtime startup.

## Q&A

1. **Is Spring Boot a replacement for Spring Framework?**
   No — Spring Boot is built on top of Spring. It uses Spring's core (IoC, DI, MVC) under the hood and adds opinionated defaults, auto-configuration, and starters.

2. **Why do we still need Spring Framework knowledge if Spring Boot exists?**
   Spring Boot hides setup, not concepts. Beans, DI, `ApplicationContext`, MVC, transactions — all Spring Framework. Debugging and advanced usage require understanding the core.

3. **Is Spring Boot faster than Spring Framework?**
   Faster to **develop**, not necessarily faster at **runtime**. Auto-configuration and classpath scanning add startup overhead. For production startup speed, GraalVM native image / Spring Native helps.

4. **What's an embedded server and why does it matter?**
   An embedded servlet container (Tomcat/Jetty/Undertow) packaged inside the app JAR. You run `java -jar app.jar` — no external server needed. Simplifies deployment, fits containerized/microservice architectures.

## Quick Reference

|Concept|Remember|
|---|---|
| Spring Framework | Core — IoC, DI, MVC. Manual config. |
| Spring Boot | Opinionated Spring + auto-config + starters + embedded server |
| Not a replacement | Boot uses Spring internally — both coexist |
| Configuration | XML (Spring) → `application.properties`/`.yml` (Boot) |
| Deployment | WAR→external server (Spring) vs JAR→embedded (Boot) |
