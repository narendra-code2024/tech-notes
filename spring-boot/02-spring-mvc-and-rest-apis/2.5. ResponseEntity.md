`ResponseEntity<T>` represents the entire HTTP response (body, status code, and headers), giving a controller full control over what is sent back. `T` is the body type (DTO, `String`, `List`, etc.).

A plain return value always produces `200 OK` with no header control; `ResponseEntity` lets you set status, body, and headers explicitly:

```java
@GetMapping("/hello")
public String hello() {
    return "Hello";                       // always 200 OK, no header control
}

@GetMapping("/hello")
public ResponseEntity<String> helloFull() {
    return ResponseEntity.ok("Hello");    // status, body, headers all controllable
}
```

---

## Creating a `ResponseEntity`

**Builder methods**: the fluent, preferred form.

```java
return ResponseEntity.ok(body);                               // 200 OK
return ResponseEntity.status(HttpStatus.CREATED).body(body);  // 201 Created
return ResponseEntity.created(uri).body(body);                // 201 + Location header
return ResponseEntity.noContent().build();                    // 204 No Content
return ResponseEntity.notFound().build();                     // 404 Not Found
return ResponseEntity.badRequest().body(errors);              // 400 Bad Request
```

**Constructor form**: still works, more verbose.

```java
return new ResponseEntity<>(body, HttpStatus.OK);
```

For a POST creating a resource, `created(uri)` sets `201` and the `Location` header in one call:

```java
@PostMapping
public ResponseEntity<ProductDto> createProduct(@RequestBody ProductDto dto) {
    ProductDto saved = productService.create(dto);
    URI location = URI.create("/api/products/" + saved.getId());
    return ResponseEntity.created(location).body(saved);
}
```

## Q&A

1. **When to use `ResponseEntity` vs returning the object directly?**
Use `ResponseEntity` when you need control over the status code (`201`, `204`, `404`) or headers (`Location`, `ETag`, cache control). For plain `200 OK` cases, returning the DTO directly is cleaner, especially with a global response wrapper (see [2.8. API Response Patterns](<2.8. API Response Patterns.md>)).

2. **How do you return `201 Created` with the new resource's URL?**
`ResponseEntity.created(uri).body(dto)`, where `uri` points to the new resource; it sets both the `201` status and the `Location` header.

3. **`noContent()` vs `ok().build()`?**
`noContent()` is `204 No Content`; `ok().build()` is `200 OK` with an empty body. Use `204` for successful operations that intentionally return nothing (DELETEs, some PUTs).
