## Program Structure

Every Java program is organized into classes, with one entry point the JVM invokes to begin execution.

### Hello World

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

Structural and JVM entry rules:

- **One public class per file**: the filename must match the public class name exactly (e.g. `HelloWorld.java`). All code lives inside classes; no free-floating statements.
- **Entry-point signature**: the JVM looks for this exact method signature.
  - `public`: callable from outside the class's package.
  - `static`: invokable without creating an instance.
  - `void`: returns nothing.
  - `main`: the entry-point name the JVM recognizes.
  - `String[] args` (or `String... args`): command-line arguments as a `String` array.
- **Alternative forms**: additional modifiers like `final` and `synchronized` may be combined (e.g. `public static final synchronized void main(String[] args)`).

> **Note**: recent previews (JEP 445 onward, Java 21+) allow a simplified instance `void main()` with no class boilerplate for small programs; the classic `public static void main(String[] args)` remains standard for production.

### Command-Line Arguments

Whatever you type after the class name becomes the `args` array:

```bash
java MyApp foo bar baz
```

```java
public class MyApp {
    public static void main(String[] args) {
        System.out.println(args.length);    // 3
        System.out.println(args[0]);        // "foo"
        System.out.println(args[1]);        // "bar"
    }
}
```

- `args` is never `null`; with no arguments it is an empty array.
- Elements are always `String`; parse to numbers yourself, e.g. `Integer.parseInt(args[0])`.
- **Parameter vs argument**: a *parameter* is the placeholder in the signature (`String[] args`); an *argument* is the actual value passed at the call site (`"foo"`).

## Statements, Expressions & Blocks

**Statements**: instructions that perform an action; each ends with `;`.

```java
int x = 5;                        // assignment
System.out.println("hello");      // expression statement
return x;                         // return
```

**Expressions**: produce a value.

```java
5 + 3                             // → 8
x > 0                             // → true/false
name.toUpperCase()                // → String
new Employee("Alice")             // → Employee instance
```

> **Note**: only assignments, increment/decrement, method calls, and object creation can stand alone as statements. Pure arithmetic (`5 + 3;`) and comparisons (`x > 0;`) are compile errors.

**Blocks**: code grouped with `{ }` (method bodies, loops, conditionals). A variable declared in a block is visible only within it.

```java
if (condition) {
    int x = 10;                   // block-scoped
    System.out.println(x);
}                                 // x goes out of scope here
```

## Identifiers & Keywords

Identifiers are the names you choose; keywords are names the language reserves.

### Identifiers

Names given to classes, methods, variables, and so on, subject to these rules:

- Can contain letters, digits, `$`, and `_`, but must start with a letter, `$`, or `_` (not a digit).
- Case-sensitive: `name` and `Name` are different.
- No other special characters, and a reserved keyword (`int`, `class`, `public`) cannot be used.

**Naming conventions**: not enforced by the compiler, but universal across the Java ecosystem.

| Entity                     | Convention               | Example                          |
| -------------------------- | ------------------------ | -------------------------------- |
| Packages                   | lowercase, dot-separated | `com.example.order`              |
| Classes / Interfaces       | `PascalCase`             | `StringBuilder`, `Runnable`      |
| Methods                    | `camelCase` (verbs)      | `getName()`, `calculateSalary()` |
| Variables / Fields         | `camelCase` (nouns)      | `userName`, `totalPrice`         |
| Constants (`static final`) | `UPPER_SNAKE_CASE`       | `MAX_RETRIES`, `DEFAULT_TIMEOUT` |
| Generic type parameters    | Single uppercase letter  | `T`, `E`, `K`, `V`               |
| Enum constants             | `UPPER_SNAKE_CASE`       | `PENDING`, `MAX_RETRIES`         |

### Keywords

Words reserved by the compiler that can never be used as identifiers.

```
abstract    continue    for           new         switch
assert      default     if            package     synchronized
boolean     do          goto*         private     this
break       double      implements    protected   throw
byte        else        import        public      throws
case        enum        instanceof    return      transient
catch       extends     int           short       try
char        final       interface     static      void
class       finally     long          strictfp    volatile
const*      float       native        super       while
```

- `goto` and `const` (marked `*`) are reserved but unused in Java.
- Literals `true`, `false`, and `null` are reserved values, not keywords.

**Contextual keywords** (restricted identifiers): reserved only within specific contexts and usable as ordinary identifiers elsewhere (though discouraged for clarity).

- **Module-related** (Java 9+): `module`, `requires`, `exports`, `opens`, `uses`, `provides`, `to`, `with`, reserved only inside `module-info.java`.
- **Local variable type inference** (Java 10+): `var`.
- **Switch expressions** (Java 14+): `yield`.
- **Records** (Java 16+): `record`.
- **Sealed classes** (Java 17+): `sealed`, `non-sealed`, `permits`.

## Comments

Java has three comment forms for explanation and documentation.

**Single-line (`//`)**: short notes or inline explanations.

```java
// TODO: validate input before processing
int x = 5; // inline comment at end of statement
```

**Multi-line (`/* ... */`)**: block notes or temporarily disabling code.

```java
/*
 * Calculates the daily interest accrued on this account.
 * Formula: (principal * rate) / 365
 */
double calculateDailyInterest(double principal, double rate) {
    return (principal * rate) / 365.0;
}
```

**Javadoc (`/** ... */`)**: processed by the `javadoc` tool to generate HTML API docs, and must sit immediately before the class, method, or field they describe.

```java
/**
 * Transfers funds from this account to the destination target.
 *
 * @param target the destination account (must not be null)
 * @param amount the transfer amount in cents
 * @return true if the transfer succeeded, false otherwise
 */
public boolean transfer(Account target, long amount) {
    return false; // stub implementation
}
```

Common Javadoc tags:

| Tag           | Use                                            |
| :------------ | :--------------------------------------------- |
| `@param`      | Describes a method parameter                   |
| `@return`     | Describes the return value                     |
| `@throws`     | Documents an exception the method may throw    |
| `@since`      | The version in which this was introduced       |
| `@deprecated` | Marks as deprecated and suggests an alternative |

## Packages & Imports

Packages provide namespaces and directory grouping; imports let you reference classes outside the current package.

```java
package com.example.orders;   // 1. Package: file must live in com/example/orders/

import java.util.List;        // 2. Import: use List instead of java.util.List
import java.util.ArrayList;
import static java.lang.Math.sqrt; // 3. Static import: use static members unqualified
import static java.lang.Math.pow;

public class OrderService {
    private List<Order> orders = new ArrayList<>();

    double hypotenuse(double a, double b) {
        return sqrt(pow(a, 2) + pow(b, 2)); // no "Math." prefix needed
    }
}
```

Key rules:

- **Package convention**: dot-separated lowercase in reversed-domain form (e.g. `com.company.project`); only one `package` statement per file.
- **Implicit imports**: classes in `java.lang` (`String`, `System`, `Math`) and classes in the same package are available without an import.
- **Wildcard imports (`*`)**: bring in every class of one package (`import java.util.*;`), but are not recursive, so they do not import sub-packages like `java.util.concurrent`.

### Common Standard Packages

| Package                | Contents                                                                                     |
| :--------------------- | :------------------------------------------------------------------------------------------- |
| `java.lang`            | Core fundamentals (`String`, `Integer`, `Object`, `System`, `Math`, `Thread`); auto-imported |
| `java.util`            | Collections and utilities (`List`, `Map`, `ArrayList`, `HashMap`, `Scanner`)                 |
| `java.io` / `java.nio` | I/O, paths, and file streaming (`File`, `InputStream`, `Path`, `Files`)                       |
| `java.time`            | Modern thread-safe date/time API (`LocalDate`, `LocalDateTime`, `Duration`)                   |
| `java.util.concurrent` | High-level concurrency utilities (`ExecutorService`, `Future`, `ConcurrentHashMap`)           |
| `java.util.stream`     | Functional stream operations and pipelines (`Stream`, `Collectors`)                           |

## Q&A

1. **Can the `String[] args` of `main` ever be `null`?**
No. With no command-line arguments the JVM passes an empty array (`new String[0]`), so `args.length` is always safe to read without a null check.

2. **Why is `$` in identifiers discouraged?**
It is legal but conventionally reserved for machine-generated code; the compiler itself uses it to name nested classes (`Outer$Inner.class`). Using it by hand risks collisions and confusion with tool-generated artifacts.

3. **Does `import java.util.*` include sub-packages?**
No. A wildcard imports only the classes directly in that package, so it gives you `List` and `Map` but not anything in `java.util.concurrent`. Sub-packages must be imported explicitly.

4. **Why must `main` be `static`?**
`static` lets the JVM invoke `main` directly on the class without first creating an instance. An instance method would force the JVM to pick and call a constructor before any entry point existed, a circular requirement that `static` removes.

| Concept                 | Remember                                                                      |
| ----------------------- | ----------------------------------------------------------------------------- |
| Filename                | Must match the public class name exactly (case-sensitive)                     |
| `main` signature        | `public static void main(String[] args)`; every word mandatory                |
| `args`                  | String array; never null, possibly empty                                      |
| Parameter vs argument   | Parameter = declaration placeholder; argument = actual value passed           |
| Identifier rules        | Letters/digits/`$`/`_`; cannot start with a digit; case-sensitive; not a keyword |
| Comments                | `//`, `/* */`, Javadoc `/** */`                                               |
| `package`               | First non-comment line; one per file                                          |
| `java.lang`             | Auto-imported; no import for `String`, `Math`, etc.                           |
| Static import           | `import static java.lang.Math.sqrt;` uses static members unqualified          |
| Wildcard import         | `java.util.*` is one level only, not recursive into sub-packages              |
