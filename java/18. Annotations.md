An annotation is a metadata tag attached to code constructs to supply declarative information without directly altering execution behavior.

Annotations are prefixed with `@` and processed at compile time or runtime by a compiler, IDE, or reflective framework. They can target classes, interfaces, fields, methods, constructors, parameters, local variables, and packages.

```java
class User {
    @Override
    public String toString() { return "User"; }
}
```

---

## Built-in Annotations

Built-in annotations enforce compile-time checks, track deprecation, or suppress compiler warnings.

- **`@Override`**: enforces that a method overrides or implements a signature in a supertype, catching typos. Cannot target static methods or constructors.
- **`@Deprecated`**: marks an element obsolete. Supports `since` (version) and `forRemoval` (flagging complete deletion).
- **`@SuppressWarnings`**: mutes specified warnings in the annotated scope. Best practice is to target the narrowest scope.
  - *Keys*: `"unchecked"`, `"deprecation"`, `"removal"` (required for `forRemoval=true` warnings since Java 9), `"rawtypes"`, `"unused"`.

```java
import java.util.List;

class Parent { void show() {} }

class Child extends Parent {
    @Override void show() {} // validates the override
}

class Legacy {
    @Deprecated(since = "17", forRemoval = true) void close() {}
}

class Demo {
    @SuppressWarnings({"unchecked", "removal"})
    void run(Object obj) {
        new Legacy().close(); // warning suppressed
        List<String> list = (List<String>) obj; // warning suppressed
    }
}
```

## Meta-Annotations

Meta-annotations configure the behavior and scope of other annotations.

- **`@Target`**: restricts the applicable Java elements. Supports `TYPE_USE` (any type usage) and `TYPE_PARAMETER` (generic declarations) since Java 8.
- **`@Retention`**: specifies metadata lifetime. Default is `CLASS` (binary-only).
- **`@Inherited`**: automatically propagates class-level annotations to subclasses. Does not apply to interfaces, methods, or fields.
- **`@Repeatable`** (Java 8): allows the same annotation to be applied more than once to one element; the compiler wraps the repeats in a designated container annotation (e.g. `@Repeatable(Schedules.class)` on `@Schedule`).

```java
import java.lang.annotation.*;

@Documented   // surfaces the annotation in generated Javadoc
@Inherited
@Target({ElementType.TYPE, ElementType.TYPE_USE})
@Retention(RetentionPolicy.RUNTIME)
@interface Auditable {}

@Auditable
class Parent {}
class Child extends Parent {} // inherits @Auditable
```

The `@Retention` policy decides how far an annotation survives the source â†’ `.class` â†’ JVM pipeline. Only `RUNTIME` is visible to reflection, so frameworks always require it.

| Retention Policy | Lives until                  | Reflection sees it | Example                           |
| :--------------- | :--------------------------- | :----------------- | :-------------------------------- |
| `SOURCE`         | Compilation (then discarded) | No                 | `@Override`, `@SuppressWarnings`  |
| `CLASS`          | `.class` file (default)      | No                 | Default compiler behavior         |
| `RUNTIME`        | Running JVM                  | Yes                | `@Deprecated`, Spring annotations |

## Custom Annotations

Custom annotations define custom metadata structures using the `@interface` keyword.

- **Rules**: element return types are restricted to primitives, `String`, `Class`, enums, annotations, or arrays of these. Elements declare default values with the `default` keyword.
- **Shorthand**: omit the key name at the call site when assigning only the `value()` element.
- **Reflection**: reading annotations at runtime requires `@Retention(RUNTIME)` and the reflection APIs.

```java
import java.lang.annotation.*;
import java.lang.reflect.Method;

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)   // RUNTIME: required for reflection to read it
@interface Retry {
    int value() default 3;            // max attempts
}

class Service {
    @Retry(5)                         // shorthand: value name omitted
    void call() { throw new RuntimeException("flaky"); }
}

// A mini-framework reads the annotation and drives behavior from it
class RetryRunner {
    static void run(Object obj, String method) throws Exception {
        Method m = obj.getClass().getDeclaredMethod(method);
        Retry r = m.getAnnotation(Retry.class);   // null if the annotation is absent
        int max = (r != null) ? r.value() : 1;
        for (int i = 1; i <= max; i++) {
            try { m.invoke(obj); return; }        // success: stop retrying
            catch (Exception e) {
                if (i == max) throw e;            // give up after max attempts
            }
        }
    }
}
```

## Q&A

1. **What is the default retention policy, and why does it matter?**
The default is `CLASS`: stored in the class file but invisible to the JVM at runtime. Reflection-driven frameworks therefore need an explicit `@Retention(RUNTIME)`.

2. **Does adding an annotation change a method's behavior?**
No. Annotations are passive metadata. Behavior changes only when a compiler, IDE, or runtime framework inspects the annotation (via reflection) and runs code in response.

3. **What is special about an element named `value()`?**
If `value()` is the only element you assign, its name can be omitted (`@MaxAttempts(5)` instead of `@MaxAttempts(value = 5)`).

4. **Can `@Inherited` make a subclass inherit annotations from a parent's methods?**
No. `@Inherited` applies only to class-level annotations. Annotations on methods, fields, constructors, or interfaces are never inherited.

5. **What are the restrictions on annotation element return types?**
Only primitives, `String`, `Class`, enums, other annotations, or arrays of these. Complex objects such as generic collections are forbidden.

## Quick Revision Cheat Sheet

| Concept             | Remember                                                                                         |
| :------------------ | :----------------------------------------------------------------------------------------------- |
| `@Override`         | Enforces signature matching against parent methods; compile-time check                           |
| `@Deprecated`       | Marks code obsolete; `since`/`forRemoval=true` triggers warnings; suppress via `"removal"`        |
| `@SuppressWarnings` | Mutes specified warnings (`unchecked`, `deprecation`, `removal`) in scope                         |
| `@Target`           | Restricts annotation scope (`METHOD`, `FIELD`, `TYPE`, `PARAMETER`, ...)                          |
| `@Retention`        | Lifecycle: `SOURCE` (discarded), `CLASS` (stored, default), `RUNTIME` (reflective)               |
| `@Inherited`        | Class-level annotations propagate to subclass types only                                         |
| `@Repeatable`       | Java 8; lets one annotation be applied multiple times via a container annotation                 |
| Custom annotation   | Declared with `@interface`; return types limited to primitives, String, Class, enums, arrays      |
