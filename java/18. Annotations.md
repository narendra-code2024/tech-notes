An annotation is a metadata tag attached to code constructs to supply declarative information without directly altering execution behavior.

Annotations are prefixed with `@` and processed at compile time or runtime by a compiler, IDE, or reflective framework.

- **Targets**: Classes, interfaces, fields, methods, constructors, parameters, local variables, and packages.

```java
class User {
    @Override
    public String toString() { return "User"; }
}
```

---

## Built-in Annotations

Built-in annotations enforce compile-time checks, track deprecation, or suppress compiler warnings.

- **`@Override`**: Enforces that a method overrides or implements a signature in a supertype, catching typos. Cannot target static methods or constructors.
- **`@Deprecated`**: Marks an element obsolete. Supports `since` (version) and `forRemoval` (flagging complete deletion).
- **`@SuppressWarnings`**: Mutes specified warnings in the annotated scope. Best practice is to target the narrowest scope.
  - *Keys*: `"unchecked"`, `"deprecation"`, `"removal"` (required for `forRemoval=true` warnings since Java 9), `"rawtypes"`, `"unused"`.

```java
import java.util.List;

class Parent { void show() {} }

class Child extends Parent {
    @Override void show() {} // validates override
}

class Legacy {
    @Deprecated(since = "17", forRemoval = true) void close() {}
}

class Demo {
    @SuppressWarnings({"unchecked", "removal"})
    void run(Object obj) {
        new Legacy().close(); // warning suppressed
        List<String> list = (List<String>) obj; // warning suppressed
    }
}
```

## Meta-Annotations

Meta-annotations configure the behavior and scope of other annotations.

- **`@Target`**: Restricts applicable Java elements. Supports `TYPE_USE` (any type invocation) and `TYPE_PARAMETER` (generic declarations) since Java 8.
- **`@Retention`**: Specifies metadata lifetime. Default is `CLASS` (binary-only).
- **`@Inherited`**: Automatically propagates class-level annotations to subclasses. Does not apply to interfaces, methods, or fields.

```java
import java.lang.annotation.*;

@Documented   // surfaces the annotation in generated Javadoc
@Inherited
@Target({ElementType.TYPE, ElementType.TYPE_USE})
@Retention(RetentionPolicy.RUNTIME)
@interface Auditable {}

@Auditable
class Parent {}
class Child extends Parent {} // inherits @Auditable
```

The `@Retention` policy decides how far an annotation survives the source â†’ `.class` â†’ JVM pipeline. Only `RUNTIME` is visible to reflection, so frameworks always require it.

| Retention Policy | Lives until                  | Reflection sees it | Example                           |
| :--------------- | :--------------------------- | :----------------- | :-------------------------------- |
| `SOURCE`         | Compilation (then discarded) | No                 | `@Override`, `@SuppressWarnings`  |
| `CLASS`          | `.class` file (default)      | No                 | Default compiler behavior         |
| `RUNTIME`        | Running JVM                   | Yes                | `@Deprecated`, Spring annotations |

## Custom Annotations

Custom annotations define custom metadata structures using the `@interface` keyword.

- **Rules**: Returns restricted to primitives, `String`, `Class`, enums, annotations, or arrays of these. Elements declare default values using the `default` keyword.
- **Shorthand**: Omit key name at call site if assigning only the `value()` element.
- **Reflection**: Accessing annotations at runtime requires `@Retention(RUNTIME)` and reflection APIs.

```java
import java.lang.annotation.*;
import java.lang.reflect.Method;

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)   // RUNTIME: required for reflection to read it
@interface Retry {
    int value() default 3;            // max attempts
}

class Service {
    @Retry(5)                         // shorthand: value name omitted
    void call() { throw new RuntimeException("flaky"); }
}

// A mini-framework reads the annotation and drives behavior from it
class RetryRunner {
    static void run(Object obj, String method) throws Exception {
        Method m = obj.getClass().getDeclaredMethod(method);
        Retry r = m.getAnnotation(Retry.class);   // null if annotation absent
        int max = (r != null) ? r.value() : 1;
        for (int i = 1; i <= max; i++) {
            try { m.invoke(obj); return; }        // success: stop retrying
            catch (Exception e) {
                if (i == max) throw e;            // give up after max attempts
            }
        }
    }
}
```

## Q&A

1. **What is the default retention policy for annotations, and why is this important to remember?**
The default is `CLASS`. These annotations are stored in class files but are unavailable to the JVM at runtime, requiring an explicit `RUNTIME` policy for reflection.

2. **Does adding an annotation to a method automatically change its execution behavior?**
No. Annotations are compile-time metadata. Behavioral changes require a compiler, IDE, or runtime framework to inspect the annotation via reflection and execute specific code.

3. **What is special about an annotation element named `value()`?**
If `value()` is the only element without a default, its name can be omitted during assignment (e.g., `@MaxAttempts(5)` instead of `@MaxAttempts(value = 5)`).

4. **Can you use `@Inherited` to make a subclass inherit an annotation from its parent's methods?**
No. `@Inherited` only applies to class-level annotations. Annotations on methods, fields, constructors, or interfaces are never inherited.

5. **What are the restrictions on the return types of annotation elements?**
They are restricted to primitives, `String`, `Class`, enums, other annotations, or arrays of these. Complex objects (e.g., generic collections) are forbidden.

## Quick Revision Cheat Sheet

| Concept             | Remember                                                                                         |
| :------------------ | :----------------------------------------------------------------------------------------------- |
| `@Override`         | Enforces signature matching against parent methods; compile-time check.                          |
| `@Deprecated`       | Marks code obsolete; `since`/`forRemoval=true` triggers warnings. Suppressed via `"removal"`.     |
| `@SuppressWarnings` | Mutes specified compiler warnings (e.g., `unchecked`, `deprecation`, `removal`) in scope.        |
| `@Target`           | Restricts annotation scope to elements (e.g., `METHOD`, `FIELD`, `TYPE`, `PARAMETER`).           |
| `@Retention`        | Sets lifecycle duration: `SOURCE` (discarded), `CLASS` (stored, default), `RUNTIME` (reflective). |
| `@Inherited`        | Class-level annotations automatically propagate to subclass types.                               |
| Custom Annotation   | Declared using `@interface`; return types restricted to primitives, String, Class, enums, arrays. |
