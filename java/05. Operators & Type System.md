## Operators

### Basic Operators

| Category       | Symbols                              | Note                                             |
| :------------- | :----------------------------------- | :----------------------------------------------- |
| **Arithmetic** | `+`, `-`, `*`, `/`, `%`              | `10 / 3 = 3` (truncates). Use `float`/`double` for decimals. |
| **Unary**      | `++`, `--`, `+`, `-`, `!`            | `++a` (update then use), `a++` (use then update) |
| **Assignment** | `=`, `+=`, `-=`, `*=`, `/=`, `%=`    | `b += 1` includes implicit cast; `b = b + 1` fails on small types. |
| **Relational** | `==`, `!=`, `>`, `<`, `>=`, `<=`     | `==` compares values (primitives) or references (objects). |
| **Logical**    | `&&`, <code>&#124;&#124;</code>, `!` | `&&`/<code>&#124;&#124;</code> short-circuit; `!` is unary. |


### Ternary

```java
// Ternary Operator (condition ? true : false)
int max = (a > b) ? a : b;

// Nested Ternary (Avoid â€” hard to read)
String grade = (score >= 90) ? "A" : (score >= 80) ? "B" : "F";
```

### Bitwise

```java
int a = 5, b = 3;   // binary: a = 0101, b = 0011
a & b;              // 1 (0001) â€” AND: 1 if both bits are 1
a | b;              // 7 (0111) â€” OR:  1 if either bit is 1
a ^ b;              // 6 (0110) â€” XOR: 1 if bits differ
~a;                 // -6       â€” NOT: flips every bit (~x = -x - 1)
```

> **Note**: 
> - `~5 = -6` because `int` is 32 bits; `~` flips all bits (including leading zeros), setting the sign bit. 
> - Shortcut: **`~x = -x - 1`**.
> - `&`, `|`, and `^` also work as non-short-circuiting logical ops on booleans.

### Shift

```java
// Examples:
int x = 5;          // binary: 0101
x << 1;             // 10 (1010) â€” Left Shift: Multiply by 2^s
x >> 1;             // 2  (0010) â€” Right Shift: Divide by 2^s
x >>> 1;            // 2  (0010) â€” Unsigned R-Shift: Logical (fills 0 on left)
```

> **Note**: 
> - **Distance Masking**: Shift distance is calculated `modulo 32` for `int` (and `modulo 64` for `long`). For example, `x << 32` is equivalent to `x << 0`.
> - **Signed vs Unsigned**: `>>` preserves the sign (fills with `1` for negatives), while `>>>` always fills with `0` on the left. 
> - **No `<<<`**: There is no `<<<` because `<<` already fills with `0` on the right.

## Type Conversion

### Typecasting
- **Widening (Implicit)**: `byte` â†’ `short`/`char` â†’ `int` â†’ `long` â†’ `float` â†’ `double`. Automatic and safe.
- **Narrowing (Explicit)**: Requires `(type)` cast. Silently discards **high-order bits** on overflow (bits wrap).

```java
double d = 9.99;
int i = (int) d;          // 9 (truncated toward zero)
long huge = 130;
byte small = (byte) huge; // -126 (bits wrap)

// Compound assignment includes implicit narrowing cast
byte b = 10;
b += 1;                   // works (compiled as b = (byte)(b + 1))
```

> **Special Cases**:
> - **Overflow**: Standard narrowing does not validate. Use `Math.toIntExact(long)` or `Math.addExact()` to throw an exception on overflow.
> - **char â†” int**: `int code = 'A';` (65); `char letter = (char) 66;` ('B').

### Type Promotion
Smaller types are **automatically promoted** pairwise (left-to-right) before operations. Floor is `int`.

| Wider Type in Expr | Result Type |
| :--- | :--- |
| `double` | `double` |
| `float` | `float` |
| `long` | `long` |
| Otherwise | `int` (even `byte` + `byte` â†’ `int`) |

> **Note**: Legal widening from `long` to `float`/`double` can silently round large values due to significant digit limits (~7 digits for `float`, ~15 for `double`).

## Boxing & Unboxing
Java handles numbers as either **Primitives** (raw values) or **Wrappers** (objects on the heap).

- **Boxing**: Primitive â†’ Wrapper.
- **Unboxing**: Wrapper â†’ Primitive.
- **Auto-boxing/unboxing**: Implicit conversion added in Java 5 to support Collections and Generics ergonomics.

```java
// Manual (Pre-Java 5)
Integer obj = Integer.valueOf(10);
int num = obj.intValue();

// Auto (Java 5+)
Integer a = 10;
int b = a;

List<Integer> list = new ArrayList<>();
list.add(20);         // implicit auto-boxing
```

> **Note**: 
> - `Integer.valueOf()` caches values from `-128` to `127`. Small values reuse objects (`==` is `true`), but larger values create new instances. **Always use `.equals()` for wrappers.**
> - Unboxing a `null` wrapper throws a `NullPointerException`.

## Q&A

1. **How does division by zero differ between integer and floating-point types?**
Integer types throw an `ArithmeticException`, while floating-point types return `Â±Infinity` or `NaN` according to IEEE 754.

2. **How can you correctly check if a double calculation resulted in NaN?**
You must use `Double.isNaN(value)`; comparing `value == Double.NaN` always returns `false` because `NaN` is technically undefined.

3. **Why do calculations like 0.1f + 0.2f not equal 0.3f?**
Binary floating-point cannot exactly represent most decimal fractions, leading to small rounding errors; use `BigDecimal` for exact financial math.

4. **How does numeric promotion affect the result of a ternary operator?**
The operator forces both branches to a common type (e.g., `Integer` and `Double` both become `double`), which can lead to unexpected unboxing or promotion.
