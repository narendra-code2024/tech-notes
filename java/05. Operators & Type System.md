## Operators

Java's operators grouped by what they do, from arithmetic through bit manipulation and type tests.

### Basic Operators

| Category       | Symbols                              | Note                                             |
| :------------- | :----------------------------------- | :----------------------------------------------- |
| **Arithmetic** | `+`, `-`, `*`, `/`, `%`              | `10 / 3 = 3` (truncates); use `float`/`double` for decimals |
| **Unary**      | `++`, `--`, `+`, `-`, `!`            | `++a` (update then use), `a++` (use then update) |
| **Assignment** | `=`, `+=`, `-=`, `*=`, `/=`, `%=`    | `b += 1` includes an implicit cast; `b = b + 1` fails on small types |
| **Relational** | `==`, `!=`, `>`, `<`, `>=`, `<=`     | `==` compares values (primitives) or references (objects) |
| **Logical**    | `&&`, <code>&#124;&#124;</code>, `!` | `&&`/<code>&#124;&#124;</code> short-circuit; `!` is unary |

### Ternary

```java
// Ternary (condition ? ifTrue : ifFalse)
int max = (a > b) ? a : b;

// Nested ternary (avoid, hard to read)
String grade = (score >= 90) ? "A" : (score >= 80) ? "B" : "F";
```

### Bitwise

```java
int a = 5, b = 3;   // binary: a = 0101, b = 0011
a & b;              // 1 (0001), AND: 1 where both bits are 1
a | b;              // 7 (0111), OR:  1 where either bit is 1
a ^ b;              // 6 (0110), XOR: 1 where bits differ
~a;                 // -6      , NOT: flips every bit (~x = -x - 1)
```

- `~5 = -6` because `int` is 32 bits: `~` flips all bits including leading zeros, setting the sign bit (shortcut: `~x = -x - 1`).
- `&`, `|`, and `^` also act as non-short-circuiting logical operators on `boolean` operands.

### Shift

```java
int x = 5;          // binary: 0101
x << 1;             // 10 (1010), left shift: multiply by 2^n
x >> 1;             // 2  (0010), signed right shift: divide by 2^n
x >>> 1;            // 2  (0010), unsigned right shift: fills 0 on the left
```

- Shift distance is taken modulo 32 for `int` (modulo 64 for `long`), so `x << 32` equals `x << 0`.
- `>>` preserves the sign (fills with the sign bit); `>>>` always fills `0`. There is no `<<<` because `<<` already fills `0` on the right.

### `instanceof`

```java
String s = "hello";
s instanceof String;    // true
s instanceof Object;    // true
s instanceof Integer;   // compile error: unrelated types

null instanceof String; // false, null is not an instance of anything

// Pattern matching (Java 16+)
if (obj instanceof String str) {
    System.out.println(str.length()); // str already cast
}
```

## Type Conversion

Java converts between numeric types implicitly when it is safe and requires an explicit cast when information could be lost.

### Typecasting

- **Widening (implicit)**: `byte → short → int → long → float → double`, and `char → int → long → float → double`. Automatic and safe. (`byte`/`short` ↔ `char` need an explicit cast.)
- **Narrowing (explicit)**: requires a `(type)` cast and silently discards high-order bits on overflow.

```java
double d = 9.99;
int i = (int) d;          // 9, truncated toward zero
long huge = 130;
byte small = (byte) huge; // -126, bits wrap

// Compound assignment includes an implicit narrowing cast
byte b = 10;
b += 1;                   // works, compiled as b = (byte)(b + 1)
```

- Narrowing does not check for overflow; use `Math.toIntExact(long)` or `Math.addExact(...)` to throw on overflow.
- `char` ↔ `int`: `int code = 'A';` gives `65`; `char letter = (char) 66;` gives `'B'`.

### Type Promotion

Smaller operands are automatically promoted (pairwise, left to right) before an operation, with `int` as the floor.

| Wider type in expression | Result type                          |
| :----------------------- | :----------------------------------- |
| `double`                 | `double`                             |
| `float`                  | `float`                              |
| `long`                   | `long`                               |
| Otherwise                | `int` (even `byte` + `byte` → `int`) |

> **Note**: widening `long` to `float`/`double` is legal but can silently round large values, since significant digits are limited (~7 for `float`, ~15 for `double`).

## Boxing & Unboxing

Boxing and unboxing convert between primitives (raw values) and their wrapper objects (on the heap), which is how primitives flow through collections and generics.

- **Boxing**: primitive → wrapper.
- **Unboxing**: wrapper → primitive.
- **Auto-boxing/unboxing**: the implicit form added in Java 5 for collection and generic ergonomics.

```java
// Manual (pre-Java 5)
Integer obj = Integer.valueOf(10);
int num = obj.intValue();

// Auto (Java 5+)
Integer a = 10;
int b = a;

List<Integer> list = new ArrayList<>();
list.add(20);         // implicit auto-boxing
```

- Unboxing a `null` wrapper throws `NullPointerException`, a hazard that hides inside arithmetic and conditions (`int x = map.get(missing);`).

> **Reference**: wrapper caching (−128…127) and why to compare wrappers with `.equals()`, see [Variables & Data Types](<04. Variables & Data Types.md>).

## Q&A

1. **How does division by zero differ between integer and floating-point types?**
Integer division by zero throws `ArithmeticException`, while floating-point division returns `±Infinity` or `NaN` per IEEE 754.

2. **How do you correctly check whether a `double` is `NaN`?**
Use `Double.isNaN(value)`; `value == Double.NaN` is always `false` because `NaN` is not equal to anything, including itself.

3. **Why does `0.1f + 0.2f` not equal `0.3f`?**
Binary floating-point cannot represent most decimal fractions exactly, so small rounding errors accumulate; use `BigDecimal` for exact (e.g. financial) math.

4. **How does numeric promotion affect a ternary operator?**
Both branches are promoted to a common type (an `Integer` and a `Double` branch both become `double`), which can trigger unexpected unboxing, and a `null` wrapper branch then throws `NullPointerException`.

## Quick Revision Cheat Sheet

| Concept             | Remember                                                  |
| ------------------- | --------------------------------------------------------- |
| Integer division    | `10 / 3 = 3` truncates; cast to `double` for decimals     |
| `++a` vs `a++`      | Pre: increment first; post: use first                     |
| `&&` vs `&`         | `&&` short-circuits, `&` evaluates both                   |
| `==` for wrappers   | Compares references; use `.equals()`; cache −128 to 127   |
| Compound assignment | `b += 1` auto-casts; `b = b + 1` does not                 |
| Type promotion      | `byte`/`short`/`char` → `int` in expressions              |
| Casting             | Widening (implicit, safe) vs narrowing (explicit, risky)  |
| Auto-boxing         | primitive ↔ wrapper; automatic since Java 5               |
| Unboxing null       | Throws `NullPointerException`                             |
| `double / 0`        | `Infinity` (no exception); `int / 0` = `ArithmeticException` |
| `float` precision   | Use `BigDecimal` for money / exact values                 |
| Bitwise NOT (`~`)   | `~x = -x - 1`                                             |
