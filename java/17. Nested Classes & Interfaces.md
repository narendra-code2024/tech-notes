A nested class is a class defined within the body of another enclosing class.

This structure logically groups classes that are tightly coupled, enhancing encapsulation by restricting access or scoping helper components.

```
                  ┌─────────────────────┐
                  │    Nested Class     │
                  └──────────┬──────────┘
                             │
              ┌──────────────┴──────────────┐
              ▼                             ▼
   ┌─────────────────────┐       ┌─────────────────────┐
   │     Inner Class     │       │ Static Nested Class │
   │    (Non-Static)     │       └─────────────────────┘
   └──────────┬──────────┘
              │
      ┌───────┼───────┐
      ▼       ▼       ▼
    Inner  Local  Anonymous
```

Java classifies nested classes into two primary divisions:
- **Static Nested Class**: A nested class declared with the `static` modifier. It behaves like a top-level class scoped inside the outer class and does not hold an implicit reference to the outer instance.
- **Inner Class**: A non-static nested class. Every instance is bound to a specific enclosing instance of the outer class and holds an implicit reference to it, allowing direct access to its instance members.

---

## Static Nested Class

Associated with the enclosing class, not an instance. It behaves like a top-level class scoped inside the outer class for packaging convenience.

```java
class Outer {
    private static int staticVar = 10;
    private int instanceVar = 20;

    static class Nested {
        void show() {
            System.out.println(staticVar);      // OK: static member
            // System.out.println(instanceVar);  // compile error: needs outer instance
        }
    }
}

Outer.Nested nested = new Outer.Nested();
nested.show();
```

- Can only access outer class's static members directly
- Can declare static and instance members; does not hold an implicit outer reference field

---
## Non-Static Inner Class

Associated with an instance of the outer class. It holds an implicit reference to the enclosing outer instance, allowing access to all its members.

```java
class Outer {
    private int x = 10;

    class Inner {
        void show() {
            System.out.println(x);           // OK: access outer's private field
            System.out.println(Outer.this.x); // OK: explicit outer reference
        }
    }
}

Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
inner.show();

Outer.Inner innerOneLine = new Outer().new Inner();
```

- Can access all outer class members (including private)
- Java 16+ allows non-static inner classes to declare static members (JEP 395)

### Non-Static Inner vs Static Nested Classes

| Feature | Non-Static Inner Class | Static Nested Class |
| :--- | :--- | :--- |
| Enclosing instance | Required (`outer.new Inner()`) | None required (`new Outer.Nested()`) |
| Outer class access | All members (including private) | Static members only |
| Static members | Allowed (Java 16+) | Allowed |
| Outer reference field | Implicitly holds `Outer.this` (`this$0`) | None |
| Common use case | Event listeners, iterators | Helper/utility classes, Builder pattern |

---

## Local Inner Class

A local inner class is an inner class defined inside the body of a method or block. Its scope is restricted to the enclosing block, and it is only visible within that block's execution.

```java
class Outer {
    void process() {
        int localVar = 10; // effectively final

        class LocalInner {
            void show() {
                System.out.println(localVar); // OK: accesses effectively final local var
            }
        }

        LocalInner li = new LocalInner();
        li.show();
    }
}
```

- Can access outer class members
- Can access local variables only if they are final or effectively final
- Rarely used in modern Java; lambda expressions are preferred for functional interfaces

---

## Anonymous Inner Class

An anonymous inner class is a class defined and instantiated in a single expression without a name. It provides a one-time implementation of an interface or extends a class, bypassing the need for a formal class declaration.

```java
// Anonymous class implementing interface
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Running");
    }
};

// Anonymous class extending abstract class
abstract class Shape {
    abstract double area();
}

Shape circle = new Shape() {
    @Override
    double area() { return Math.PI * 5 * 5; }
};

// Anonymous class extending concrete class
Thread t = new Thread() {
    @Override
    public void run() {
        System.out.println("Custom thread");
    }
};
```

### Anonymous Class vs Lambda

```java
// Anonymous class: verbose, generates a separate class file
Runnable r1 = new Runnable() {
    public void run() { System.out.println("Run"); }
};

// Lambda: concise, dynamically linked at runtime
Runnable r2 = () -> System.out.println("Run");
```

| Feature | Anonymous Inner Class | Lambda |
| :--- | :--- | :--- |
| Target types | Any interface or class extension | Functional interfaces only (single abstract method) |
| `this` reference | Refers to the anonymous class instance | Refers to the enclosing class instance |
| Instance state | Can declare fields and instance initializers | Cannot declare instance fields |
| Class file generation | Generates a physical `.class` file | Linked dynamically at runtime via `invokedynamic` |
| Primary use case | Multi-method implementations or stateful callbacks | Single-method functional behaviors |

## Nested Interfaces and Abstract Classes

### Nested Interfaces

An interface defined inside a class or another interface is implicitly static and public.

```java
class Outer {
    interface Callback { // implicitly static
        void onComplete();
    }
}

class MyCallback implements Outer.Callback {
    public void onComplete() { System.out.println("Done"); }
}

interface Enclosing {
    interface Inner { // implicitly public static
        void process();
    }
}
```

> **Note**: A real-world example is `Map.Entry<K,V>`, which is a nested interface within the `Map` interface.

### Nested Abstract Classes

An abstract class defined inside another class can be either static or non-static.

```java
class Outer {
    // Member inner abstract class
    abstract class InnerAbstract {
        abstract void process();
        void log() { System.out.println("Logging"); }
    }

    class ConcreteInner extends InnerAbstract {
        void process() { System.out.println("Processing"); }
    }

    // Static nested abstract class
    static abstract class Helper {
        abstract void help();
    }
}

class MyHelper extends Outer.Helper {
    void help() { System.out.println("Helping"); }
}
```

## Q&A

1. **How does access control work between a nested class and its outer class, and how does the JVM handle it under the hood?**
Nested and outer classes have full mutual access to each other's private members. Prior to Java 11, the compiler generated synthetic package-private accessor methods (e.g., `access$000`) to bypass JVM access restrictions; Java 11+ introduced Nest-Based Access Control (JEP 181), defining them as "nestmates" in the class file and allowing direct private access at the JVM level without bridge methods.

2. **Why is the serialization of non-static inner classes discouraged, and how does it affect the enclosing outer class?**
Non-static inner classes hold an implicit reference (via the synthetic `this$0` field) to their enclosing outer instance. Serializing an instance of a non-static inner class forces the serialization of the entire outer instance; if the outer class does not implement `Serializable`, a `NotSerializableException` is thrown.

3. **Why does double-brace initialization (which creates anonymous inner classes) cause memory leaks and performance overhead?**
Double-brace initialization (e.g., `new ArrayList<String>() {{ add("val"); }}`) creates an anonymous inner class holding an implicit outer reference (via the synthetic `this$0` field). If this collection is stored in a long-lived context, it prevents the enclosing outer object from being garbage-collected, and creates a separate `.class` file for every site.

4. **Why must local variables accessed by local or anonymous inner classes be final or effectively final?**
Local classes access local variables by capturing a copy of them in compiler-generated fields to ensure the variables remain accessible after the enclosing method returns. To prevent desynchronization between the original variable and the captured copy, Java requires them to be final or effectively final (never reassigned).

5. **What is the difference in JVM class generation and execution between anonymous inner classes and lambda expressions?**
Each anonymous inner class generates a separate physical `.class` file (e.g., `Outer$1.class`) at compile time, increasing JAR size and class-loading overhead. Lambda expressions do not generate class files; they compile to an `invokedynamic` instruction that dynamically bootstrap-links to a private method, reducing memory footprint and startup time.

## Quick Revision Cheat Sheet

| Concept                 | Remember                                                                                       |
| :---------------------- | :--------------------------------------------------------------------------------------------- |
| Non-static inner        | Associated with outer instance; holds implicit `this$0` reference field; can cause memory leaks. |
| Static nested           | Associated with outer class; does not hold outer instance reference; behaves like top-level.   |
| Local inner             | Declared inside a method; scoped to method execution; accesses final/effectively final vars.   |
| Anonymous inner         | No name; defined and instantiated in one expression; cannot define constructors (initializers only). |
| Anonymous vs Lambda     | Lambdas target functional interfaces (invokedynamic); anonymous classes target any interface/class. |
| Nested interface        | Implicitly static and public; e.g., `Map.Entry` inside `Map`.                                  |
| Nestmates (JEP 181)     | Java 11+ allows mutual private access natively without compiler-generated synthetic accessors. |
| Compilation artifact   | Anonymous classes compile to `Outer$1.class`; inner class constructors receive the enclosing instance to initialize the synthetic `this$0` field. |
| Double-brace leak       | Creates anonymous inner class instances holding outer references; prevents GC of enclosing objects. |
