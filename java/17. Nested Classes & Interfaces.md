A nested class is a class defined within the body of another enclosing class.

This structure logically groups tightly coupled classes, improving encapsulation by restricting access or scoping helper components.

```
                  ┌─────────────────────┐
                  │    Nested Class     │
                  └──────────┬──────────┘
                             │
              ┌──────────────┴──────────────┐
              ▼                             ▼
   ┌─────────────────────┐       ┌─────────────────────┐
   │     Inner Class     │       │ Static Nested Class │
   │    (Non-Static)     │       └─────────────────────┘
   └──────────┬──────────┘
              │
      ┌───────┼───────┐
      ▼       ▼       ▼
    Inner  Local  Anonymous
```

Java classifies nested classes into two primary divisions:

- **Static Nested Class**: declared with `static`. It behaves like a top-level class scoped inside the outer class and holds no implicit reference to an outer instance.
- **Inner Class**: a non-static nested class. Every instance is bound to a specific enclosing instance and holds an implicit reference to it, allowing direct access to its instance members.

---

## Static Nested Class

Associated with the enclosing class, not an instance. It behaves like a top-level class scoped inside the outer class for packaging convenience.

```java
class Outer {
    private static int staticVar = 10;
    private int instanceVar = 20;

    static class Nested {
        void show() {
            System.out.println(staticVar);       // OK: static member
            // System.out.println(instanceVar);   // compile error: needs an outer instance
        }
    }
}

Outer.Nested nested = new Outer.Nested();
nested.show();
```

- Can access the outer class's static members directly only.
- Can declare static and instance members; holds no implicit outer-reference field.

---

## Non-Static Inner Class

Associated with an instance of the outer class. It holds an implicit reference to the enclosing instance, allowing access to all its members.

```java
class Outer {
    private int x = 10;

    class Inner {
        void show() {
            System.out.println(x);            // OK: access outer's private field
            System.out.println(Outer.this.x); // OK: explicit outer reference
        }
    }
}

Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
inner.show();

Outer.Inner innerOneLine = new Outer().new Inner();
```

- Can access all outer class members (including private).
- Java 16+ allows non-static inner classes to declare static members (JEP 395).

### Non-Static Inner vs Static Nested Classes

| Feature               | Non-Static Inner Class                   | Static Nested Class                      |
| :-------------------- | :--------------------------------------- | :--------------------------------------- |
| Enclosing instance    | Required (`outer.new Inner()`)           | None required (`new Outer.Nested()`)     |
| Outer class access    | All members (including private)          | Static members only                      |
| Static members        | Allowed (Java 16+)                       | Allowed                                  |
| Outer reference field | Implicitly holds `Outer.this` (`this$0`) | None                                     |
| Common use case       | Event listeners, iterators               | Helper/utility classes, Builder pattern  |

---

## Local Inner Class

A local inner class is defined inside a method or block. Its scope is restricted to that block, and it is visible only during the block's execution.

```java
class Outer {
    void process() {
        int localVar = 10; // effectively final

        class LocalInner {
            void show() {
                System.out.println(localVar); // OK: accesses an effectively final local
            }
        }

        LocalInner li = new LocalInner();
        li.show();
    }
}
```

- Can access outer class members.
- Can access local variables only if they are final or effectively final.
- Rarely used in modern Java; lambdas are preferred for functional interfaces.

---

## Anonymous Inner Class

An anonymous inner class is defined and instantiated in a single expression without a name. It provides a one-time implementation of an interface or a one-off extension of a class, with no formal class declaration.

```java
// Anonymous class implementing an interface
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Running");
    }
};

// Anonymous class extending an abstract class
abstract class Shape {
    abstract double area();
}

Shape circle = new Shape() {
    @Override
    double area() { return Math.PI * 5 * 5; }
};

// Anonymous class extending a concrete class
Thread t = new Thread() {
    @Override
    public void run() {
        System.out.println("Custom thread");
    }
};
```

### Anonymous Class vs Lambda

```java
// Anonymous class: verbose, generates a separate class file
Runnable r1 = new Runnable() {
    public void run() { System.out.println("Run"); }
};

// Lambda: concise, dynamically linked at runtime
Runnable r2 = () -> System.out.println("Run");
```

| Feature               | Anonymous Inner Class                              | Lambda                                              |
| :-------------------- | :------------------------------------------------ | :-------------------------------------------------- |
| Target types          | Any interface or class extension                  | Functional interfaces only (single abstract method) |
| `this` reference      | The anonymous class instance                      | The enclosing class instance                        |
| Instance state        | Can declare fields and instance initializers      | Cannot declare instance fields                      |
| Class file generation | Generates a physical `.class` file                | Linked dynamically at runtime via `invokedynamic`   |
| Primary use case      | Multi-method implementations or stateful callbacks | Single-method functional behaviors                  |

## Nested Interfaces and Abstract Classes

Beyond classes, interfaces and abstract classes can also be nested, each with their own scoping rules.

### Nested Interfaces

An interface defined inside a class or another interface is implicitly static and public.

```java
class Outer {
    interface Callback { // implicitly static
        void onComplete();
    }
}

class MyCallback implements Outer.Callback {
    public void onComplete() { System.out.println("Done"); }
}

interface Enclosing {
    interface Inner { // implicitly public static
        void process();
    }
}
```

> **Note**: a real-world example is `Map.Entry<K,V>`, a nested interface within the `Map` interface.

### Nested Abstract Classes

An abstract class defined inside another class can be either static or non-static.

```java
class Outer {
    // Member inner abstract class
    abstract class InnerAbstract {
        abstract void process();
        void log() { System.out.println("Logging"); }
    }

    class ConcreteInner extends InnerAbstract {
        void process() { System.out.println("Processing"); }
    }

    // Static nested abstract class
    static abstract class Helper {
        abstract void help();
    }
}

class MyHelper extends Outer.Helper {
    void help() { System.out.println("Helping"); }
}
```

## Q&A

1. **How does access between a nested class and its outer class work under the hood?**
They have full mutual access to each other's private members. Before Java 11 the compiler generated synthetic package-private accessors (e.g. `access$000`) to bypass JVM access checks; Java 11+ uses Nest-Based Access Control (JEP 181), declaring them "nestmates" in the class file so private access works directly at the JVM level without bridge methods.

2. **Why is serializing a non-static inner class discouraged?**
A non-static inner class holds an implicit reference (the synthetic `this$0` field) to its enclosing instance. Serializing the inner instance forces serialization of the whole outer instance, and if the outer class isn't `Serializable`, it throws `NotSerializableException`.

3. **Why does double-brace initialization cause memory leaks and overhead?**
`new ArrayList<String>() {{ add("val"); }}` creates an anonymous inner class holding an implicit `this$0` reference to the enclosing instance. Stored in a long-lived context, it prevents the enclosing object from being garbage-collected, and it emits a separate `.class` file per use site.

4. **Why must captured local variables be final or effectively final?**
A local or anonymous class captures a copy of each local into a compiler-generated field so it stays accessible after the method returns. To prevent the copy and the original from diverging, Java requires the variable to be final or effectively final (never reassigned).

5. **How do anonymous classes and lambdas differ in JVM class generation?**
Each anonymous class compiles to a separate `.class` file (e.g. `Outer$1.class`), adding JAR size and class-loading cost. A lambda emits no class file; it compiles to an `invokedynamic` instruction that bootstrap-links to a private method, reducing footprint and startup time.

## Quick Revision Cheat Sheet

| Concept              | Remember                                                                                       |
| :------------------- | :--------------------------------------------------------------------------------------------- |
| Non-static inner     | Bound to an outer instance; holds implicit `this$0`; can cause memory leaks                     |
| Static nested        | Bound to the outer class; no outer-instance reference; behaves like top-level                   |
| Local inner          | Declared in a method; scoped to that block; captures final/effectively-final locals             |
| Anonymous inner      | No name; defined and instantiated in one expression; no constructors (initializers only)        |
| Anonymous vs lambda  | Lambdas target functional interfaces (`invokedynamic`); anonymous classes target any type        |
| Nested interface     | Implicitly static and public; e.g. `Map.Entry` inside `Map`                                     |
| Nestmates (JEP 181)  | Java 11+ allows mutual private access natively, without synthetic accessors                     |
| Compilation artifact | Anonymous classes compile to `Outer$1.class`; inner constructors receive the enclosing instance  |
| Double-brace leak    | Anonymous instance holds an outer reference, preventing GC of the enclosing object               |
