## Variables

### Declaration and Initialization

```java
// Declaration only
int count;

// Declaration + initialization
int count = 0;
String name = "Alice";

// Multiple in one statement
int x = 1, y = 2, z = 3;

// Chained assignment
int a, b;
a = b = 10;
```

### Scope and Lifetime

Scopes control where a variable is visible and how long it lives.

| Scope              | Where declared                | Lifetime                             |
| ------------------ | ----------------------------- | ------------------------------------ |
| **Local**          | Inside a method / block       | Until the block exits                |
| **Instance**       | Inside a class (not `static`) | Same as the object                   |
| **Static (class)** | Inside a class with `static`  | Same as the class (program lifetime) |

```java
class Employee {
    static int count;           // static — lives for program duration
    String name;                // instance — lives with the object

    void greet() {
        int tmp;                // local — lives only in greet()
        // System.out.println(tmp); // ❌ COMPILE ERROR — local vars must be initialized before read
    }
}
```

> **Note:** Instance and static variables **auto-initialize** to defaults. Local variables do NOT; the compiler will error if you try to read an uninitialized local.

### `final` Keyword

Makes a variable's reference unchangeable after initial assignment.

```java
final int MAX = 100;
MAX = 200;                 // ❌ COMPILE ERROR — can't reassign

final List<String> items = new ArrayList<>();
items.add("apple");        // ✅ allowed — modifying the object, not the reference
items = new ArrayList<>(); // ❌ COMPILE ERROR — can't reassign
```

- `final` variables must be assigned **exactly once** (at declaration or in constructor for instance fields)
- `final` prevents **reference reassignment**, NOT object mutation

> **final** keyword explained in detail in [18. Important Keywords (Deep Dive)](<18. Important Keywords (Deep Dive).md>).

### `var` Keyword (Java 10+)

```java
var name = "Alice";                    // String
var count = 42;                        // int
var list = new ArrayList<String>();    // ArrayList<String>

// Can't use for these:
var x;                                 // ❌ no initializer
var fn = null;                         // ❌ null has no type
public var name = "...";               // ❌ not for instance/static fields
public void go(var x) { }              // ❌ not for method parameters
```

**Rules**:

- Compiler infers the type from the right-hand side
- Must have initializer in the declaration
- Can only use for local variables (inside method / block)
- Initializer must not be `null`
- Array shortcuts `var arr = {1, 2, 3};` are not allowed
	- Must use the standard format  `var arr = new int[]{1, 2, 3};`
- Works with loops: `for (var i = 0; i < 10; i++) { }`
- Allowed in **lambda parameters** since Java 11: `(var x, var y) -> x + y`

**Best practice**:
- use `var` when the type is obvious from the right-hand side
	`var list = new ArrayList<Order>();`

- Avoid when it obscures the type
	`var result = service.process(input);`

## Primitive Data Types

Java has exactly 8 primitive types:

| Type      | Size                                      | Range                                  | Default    |
| --------- | ----------------------------------------- | -------------------------------------- | ---------- |
| `byte`    | 1 byte (8 bits)                           | -128 to 127                            | `0`        |
| `short`   | 2 bytes (16 bits)                         | -32,768 to 32,767                      | `0`        |
| `int`     | 4 bytes (32 bits)                         | -2,147,483,648 to 2,147,483,647        | `0`        |
| `long`    | 8 bytes (64 bits)                         | -9.2 × 10¹⁸ to 9.2 × 10¹⁸              | `0L`       |
| `float`   | 4 bytes (32 bits)                         | IEEE 754 single precision (~7 digits)  | `0.0f`     |
| `double`  | 8 bytes (64 bits)                         | IEEE 754 double precision (~15 digits) | `0.0`      |
| `char`    | 2 bytes (16 bits)                         | 0 to 65,535 (Unicode unsigned)         | `'\u0000'` |
| `boolean` | implementation-defined (typically 1 byte) | `true` / `false`                       | `false`    |

**Key points**:

- `char` is **unsigned**, all other numeric types are signed
- `char` is a numeric type under the hood, can do arithmetic on it
- `int` is Java's default integer type (literal `42` is an `int`)
- `double` is Java's default floating-point type (literal `3.14` is a `double`)
- `float` literals need the suffix `f` or `F`: `3.14f`
- `long` literals larger than `int` range need the suffix `L` or `l`: `10000000000L`
- **Lowercase `l` vs uppercase `L`**: always use `L`. Lowercase looks like the digit `1`.

```java
int i = 100;                    // int by default
long l = 10_000_000_000L;       // L (or l) — long; without L, literal is int and overflows
float f = 3.14f;                // f (or F) — float; without f, 3.14 is double and won't fit
double d = 3.14;                // double by default
double d2 = 3.14d;              // d (or D) — optional; double is the default for floating-point
char grade = 'A';
boolean active = true;
```

## Literals

Literals are values you can write directly in code.

- **Integer (int & long)**: `42`, `0x2A`, `0b101010`, `052`, `10L`
- **Floating-Point (float & double)**: `3.14`, `0.234`, `5.6f`
- **Character**: `'A'`, `'\n'` (Escape sequences), `'\u0041'` (Unicode sequences)
- **String**: `"hello"`
- **Boolean**: `true`, `false`
- **Null**: `null`

### Integer Literals

```java
int decimal = 42;          // base 10
int hex     = 0x2A;        // base 16 — prefix 0x or 0X
int binary  = 0b101010;    // base 2 (Java 7+) — prefix 0b or 0B
int octal   = 052;         // base 8 — prefix 0 (rarely used, easy to mistake)
```

By default, these are `int`. Use `L` or `l` for `long` (e.g., 100L`).

### Character Literals

Single-quoted, single character OR escape sequence OR Unicode sequence:

```java
char c1 = 'A';           // regular char
char c2 = '\n';          // newline escape
char c3 = '\u0041';      // Unicode escape — same as 'A' (U+0041)
char c4 = '\t';          // tab
char c5 = '\\';          // literal backslash
```

### Escape sequences

| Escape   | Meaning                          |
| -------- | -------------------------------- |
| `\n`     | Newline                          |
| `\t`     | Tab                              |
| `\r`     | Carriage return                  |
| `\\`     | Literal backslash                |
| `\'`     | Literal single quote             |
| `\"`     | Literal double quote             |
| `\b`     | Backspace                        |
| `\f`     | Form feed                        |
| `\0`     | Null character                   |
| `\uXXXX` | Unicode character (4 hex digits) |

### Underscore separators (Java 7+)

Improves readability of long numbers:

```java
int million = 1_000_000;
long bigNumber = 1_000_000_000_000L;
int hexMask = 0xFF_00_FF_00;
int binary = 0b1010_0101_1100_0011;
```

Rules:

- Only BETWEEN digits — `1_000` ✅, `_1000` ❌, `1000_` ❌
- Compiler strips underscores before parsing
- Doesn't work in input strings: `Integer.parseInt("1_000")` ❌ fails

## Wrapper Classes

For each primitive, Java has an **object wrapper class**

|Primitive|Wrapper|
|---|---|
|`byte`|`Byte`|
|`short`|`Short`|
|`int`|`Integer`|
|`long`|`Long`|
|`float`|`Float`|
|`double`|`Double`|
|`char`|`Character`|
|`boolean`|`Boolean`|

**When to use wrappers**:

- Collections can't store primitives `List<int>`, use `List<Integer>`
- Primitives can't be `null`; wrappers can `Integer x = null`
- Generics type parameters are always reference types, `<T>` needs a class, not a primitive

```java
// Boxing (primitive → wrapper)
int primitive = 42;
Integer wrapped = primitive;      // auto-boxing (Java 5+)

// Unboxing (wrapper → primitive)
Integer wrapped2 = 42;
int primitive2 = wrapped2;        // auto-unboxing

// Collections — must use wrappers
List<Integer> nums = new ArrayList<>();
nums.add(42);                     // auto-boxing: int → Integer
int first = nums.get(0);          // auto-unboxing: Integer → int
```

## Non-Primitive Data Types

Everything that isn't a primitive is a **reference type** i.e., variable holds a reference (pointer) to an object on the heap.

| Type           | Description                                                           | Chapter                                                                               |
| -------------- | --------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| **String**     | Immutable character sequence                                          | [9. Strings](<09. Strings.md>)                                                        |
| **Arrays**     | Fixed-size ordered collection of same-type values                     | [8. Arrays](<08. Arrays.md>)                                                          |
| **Classes**    | User-defined types — blueprint for objects                            | [12. Classes, Objects & OOP Foundations](<12. Classes, Objects & OOP Foundations.md>) |
| **Interfaces** | Contract of behaviors — no state, implementations provided by classes | [16. Abstraction & Interfaces](<16. Abstraction & Interfaces.md>)                     |
| **Enums**      | Fixed set of named constants — can carry state and behavior           | [21. Enums](<21. Enums.md>)                                                           |

```java

int[] numbers = {1, 2, 3};                   // array
Employee emp = new Employee("Alice");         // class instance
```

## Q&A

1. **How does the JVM's treatment of local variables differ from instance/static variables regarding initialization?**
   Instance and static variables are automatically initialized to default values (0, null, false) when the class or object is loaded. However, local variables are stored on the stack and **must be explicitly initialized** before use; the compiler enforces this strictly to prevent non-deterministic behavior from reading "garbage" data.

2. **Why do numeric literals in Java require suffixes like `L` or `f`?**
   By default, Java treats all whole-number literals as `int` and all decimal literals as `double`. Suffixes like `L` (for `long`) and `f` (for `float`) are required to prevent overflow or precision-loss errors that occur when the default literal type is larger than the target variable or exceeds its own capacity (e.g., a literal `10_000_000_000` is too large to be an `int`).

3. **Does marking an object reference as `final` ensure the object's state is immutable?**
   No. The `final` modifier only ensures **reference immutability**, meaning the variable cannot be reassigned to point to a different object. The state of the object itself (e.g., adding elements to a `final List`) can still be modified unless the class was specifically designed to be immutable, like `String` or `Integer`.

4. **What is the "Diamond Operator" trap when using `var` for local variable type inference?**
   When combining `var` with the diamond operator (e.g., `var list = new ArrayList<>();`), the compiler has no target type to infer from and defaults to `ArrayList<Object>`. To maintain type safety, you must either specify the type in the diamond operator (`new ArrayList<String>()`) or use an explicit type declaration instead of `var`.

## Quick Revision Cheat Sheet

| Concept                     | Remember                                                                         |
| --------------------------- | -------------------------------------------------------------------------------- |
| Local var default           | None, must initialize before use                                                 |
| Instance/static var default | 0, 0.0, false, null (auto-initialized)                                           |
| `final`                     | Reassignment forbidden; object mutation still allowed                            |
| `var` (Java 10+)            | Local-only type inference; no initializer = error                                |
| Primitives                  | 8 types — `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean`   |
| `int` default               | Integer literals are `int` unless suffixed `L`                                   |
| `double` default            | Floating-point literals are `double` (optional `d`/`D`); use `f`/`F` for `float` |
| `char`                      | 16-bit unsigned                                                                  |
| Integer formats             | Decimal, `0x` hex, `0b` binary, `0` octal                                        |
| Underscore separators       | `1_000_000` — between digits only (Java 7+)                                      |
| Wrappers                    | `Integer`, `Long`, `Double`, etc. — needed for Collections, nullability          |
| Reference types             | String, Arrays, Classes, Interfaces, Enums — variables hold pointers             |
