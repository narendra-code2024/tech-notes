## Variables

A variable is a container that holds data during program execution.

```java
int x;                     // declaration only
int count = 10;            // declaration and initialization
int a, b;                  // declaring multiple variables
a = b = 5;                 // chained assignment (both get 5)
int i = 1, j = 2;          // declaring and initializing multiple in one statement
```

### Scope and Lifetime

Scopes control where a variable is visible and how long it lives.

| Scope              | Where declared                | Lifetime                             |
| ------------------ | ----------------------------- | ------------------------------------ |
| **Local**          | Inside a method / block       | Until the block exits                |
| **Instance**       | Inside a class (not `static`) | Same as the object                   |
| **Static (class)** | Inside a class with `static`  | Same as the class (program lifetime) |

```java
class Employee {
    static int count;           // static — lives for program duration
    String name;                // instance — lives with the object

    void greet() {
        int tmp;                // local — lives only in greet()
        // System.out.println(tmp); // ❌ COMPILE ERROR — local vars must be initialized before read
    }
}
```

> **Note:** Instance and static variables **auto-initialize** to defaults. Local variables do NOT; the compiler will error if you try to read an uninitialized local.

### `final` Keyword

Makes a variable's reference unchangeable after initial assignment.

```java
final int MAX = 100;
MAX = 200;                 // ❌ COMPILE ERROR — can't reassign

final List<String> items = new ArrayList<>();
items.add("apple");        // ✅ allowed — modifying the object, not the reference
items = new ArrayList<>(); // ❌ COMPILE ERROR — can't reassign
```

- `final` variables must be assigned **exactly once** (at declaration or in constructor for instance fields)
- `final` prevents **reference reassignment**, NOT object mutation

> **final** keyword explained in detail in [18. Important Keywords (Deep Dive)](<18. Important Keywords (Deep Dive).md>).

### `var` Keyword (Java 10+)

### `var` Keyword (Java 10+)

Enables **Local Variable Type Inference**, letting the compiler deduce the type from the right-hand side initializer.

```java
// Valid uses
var name = "Alice";                 // infers String
var count = 42;                     // infers int
var list = new ArrayList<String>(); // infers ArrayList<String>

// ❌ Invalid uses (Compile Errors)
var x;                              // no initializer to infer type
var val = null;                     // null has no distinct type
public var name = "Bob";            // var is restricted to local variables (not fields)
public void greet(var x) {}         // not allowed in method parameters
```

**Key Constraints & Best Practices**:
- **Applicability**: Restricted to local variables inside methods or blocks. Cannot be used for class fields or method signatures.
- **Initializers**: An active initializer is mandatory and **must not be null** (unless cast).
- **Arrays**: Inline array shortcuts like `var a = {1, 2};` are illegal; use `var a = new int[]{1, 2};`.
- **Lambda Parameters**: Allowed in lambda expressions since Java 11: `(var a, var b) -> a + b`.
- **Best Practice**: Use `var` only when the inferred type is highly obvious (e.g., `var list = new ArrayList<User>();`), avoiding it when it obscures code readability (e.g., `var result = service.fetch();`).

## Primitive Data Types

Java has exactly 8 primitive types:

| Type      | Size                                      | Range                                  | Default    |
| :-------- | :---------------------------------------- | :------------------------------------- | :--------- |
| `byte`    | 1 byte (8 bits)                           | -128 to 127                            | `0`        |
| `short`   | 2 bytes (16 bits)                         | -32,768 to 32,767                      | `0`        |
| `int`     | 4 bytes (32 bits)                         | -2,147,483,648 to 2,147,483,647        | `0`        |
| `long`    | 8 bytes (64 bits)                         | -9.2 × 10¹⁸ to 9.2 × 10¹⁸              | `0L`       |
| `float`   | 4 bytes (32 bits)                         | IEEE 754 single precision (~7 digits)  | `0.0f`     |
| `double`  | 8 bytes (64 bits)                         | IEEE 754 double precision (~15 digits) | `0.0`      |
| `char`    | 2 bytes (16 bits)                         | 0 to 65,535 (Unicode unsigned)         | `'\u0000'` |
| `boolean` | implementation-defined (typically 1 byte) | `true` / `false`                       | `false`    |

### Primitive Characteristics
While all numeric types are signed, `char` is uniquely **unsigned** (representing 16-bit Unicode points) and supports standard arithmetic under the hood.

```java
int i = 100;                  // int: default whole-number type
long l = 10_000_000_000L;     // long: requires 'L' suffix to prevent overflow (always uppercase 'L')
float f = 3.14f;              // float: requires 'f' suffix; default decimals are double
double d = 3.14;              // double: default floating-point type (optional 'd')
char grade = 'A';             // char: unsigned 16-bit Unicode character literal
boolean active = true;        // boolean: true or false only
```

## Literals

Literals are raw values written directly in the source code.

```java
// 1. Integer Literals (Default: int)
int dec = 42;
int hex = 0x2A;            // base 16 (prefix 0x or 0X)
int bin = 0b101010;        // base 2 (prefix 0b or 0B, Java 7+)
int oct = 052;             // base 8 (prefix 0 - avoid, easy to mistake)
long val = 100_000_000L;   // 'L' suffix for long; underscores improve readability

// 2. Floating-Point Literals (Default: double)
double d = 3.14;
float f = 5.6f;            // 'f' or 'F' suffix is mandatory for float

// 3. Character Literals (Single-quoted)
char letter = 'A';
char tab = '\t';           // escape sequence
char unicode = '\u0041';   // Unicode escape for 'A' (U+0041)

// 4. String, Boolean & Null Literals
String s = "hello";        // double-quoted
boolean active = true;     // true or false
Object obj = null;         // null literal
```

### Escape Sequences

| Escape   | Meaning                                                 |
| :------- | :------------------------------------------------------ |
| `\n`     | Newline                                                 |
| `\t`     | Tab                                                     |
| `\r`     | Carriage return                                         |
| `\\`     | Literal backslash                                       |
| `\'`     | Literal single quote                                    |
| `\"`     | Literal double quote                                    |
| `\b`     | Backspace                                               |
| `\f`     | Form feed                                               |
| `\0`     | Null character                                          |
| `\uXXXX` | Unicode character (4 hex digits)                        |

### Underscore Separators (Java 7+)
Allows placing underscores (`_`) between digits to improve number readability:
- **Rule**: Allowed **only between digits** (e.g., `1_000` is valid, but `_1000` or `1000_` are compile errors).
- **String Parsing**: Does not work inside input strings; e.g., `Integer.parseInt("1_000")` will throw a `NumberFormatException`.

## Wrapper Classes

For each primitive, Java has an **object wrapper class**

| Primitive | Wrapper     |
| :-------- | :---------- |
| `byte`    | `Byte`      |
| `short`   | `Short`     |
| `int`     | `Integer`   |
| `long`    | `Long`      |
| `float`   | `Float`     |
| `double`  | `Double`    |
| `char`    | `Character` |
| `boolean` | `Boolean`   |

**When to use wrappers**:

- Collections can't store primitives `List<int>`, use `List<Integer>`
- Primitives can't be `null`; wrappers can `Integer x = null`
- Generics type parameters are always reference types, `<T>` needs a class, not a primitive

```java
// Boxing (primitive → wrapper)
int primitive = 42;
Integer wrapped = primitive;      // auto-boxing (Java 5+)

// Unboxing (wrapper → primitive)
Integer wrapped2 = 42;
int primitive2 = wrapped2;        // auto-unboxing

// Collections — must use wrappers
List<Integer> nums = new ArrayList<>();
nums.add(42);                     // auto-boxing: int → Integer
int first = nums.get(0);          // auto-unboxing: Integer → int
```

## Non-Primitive Data Types

Everything that isn't a primitive is a **reference type** i.e., variable holds a reference (pointer) to an object on the heap.

| Type           | Description                                                           | Chapter                                                                               |

| **String**     | Immutable character sequence                                          | [9. Strings](<09. Strings.md>)                                                        |

```java

Employee emp = new Employee("Alice");         // class instance
```

## Q&A

1. **How does the JVM's treatment of local variables differ from instance/static variables regarding initialization?**
Instance and static variables are automatically initialized to default values (0, null, false) when the class or object is loaded. However, local variables are stored on the stack and **must be explicitly initialized** before use; the compiler enforces this strictly to prevent non-deterministic behavior from reading "garbage" data.

2. **Why do numeric literals in Java require suffixes like `L` or `f`?**
By default, Java treats all whole-number literals as `int` and all decimal literals as `double`. Suffixes like `L` (for `long`) and `f` (for `float`) are required to prevent overflow or precision-loss errors that occur when the default literal type is larger than the target variable or exceeds its own capacity (e.g., a literal `10_000_000_000` is too large to be an `int`).

3. **Does marking an object reference as `final` ensure the object's state is immutable?**
No. The `final` modifier only ensures **reference immutability**, meaning the variable cannot be reassigned to point to a different object. The state of the object itself (e.g., adding elements to a `final List`) can still be modified unless the class was specifically designed to be immutable, like `String` or `Integer`.

4. **What is the "Diamond Operator" trap when using `var` for local variable type inference?**
When combining `var` with the diamond operator (e.g., `var list = new ArrayList<>();`), the compiler has no target type to infer from and defaults to `ArrayList<Object>`. To maintain type safety, you must either specify the type in the diamond operator (`new ArrayList<String>()`) or use an explicit type declaration instead of `var`.
