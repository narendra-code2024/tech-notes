## Variables

A variable is a named container that holds data during program execution.

```java
int x;                     // declaration only
int count = 10;            // declaration and initialization
int a, b;                  // declaring multiple variables
a = b = 5;                 // chained assignment (both get 5)
int i = 1, j = 2;          // declare and initialize multiple in one statement
```

### Scope and Lifetime

Scope controls where a variable is visible and how long it lives. Java has three kinds; **local** variables are the focus here, while **instance** and **static** fields are class members previewed below.

| Scope              | Where declared                | Lifetime                             |
| ------------------ | ----------------------------- | ------------------------------------ |
| **Local**          | Inside a method / block       | Until the block exits                |
| **Instance**       | Inside a class (not `static`) | Same as the object                   |
| **Static (class)** | Inside a class with `static`  | Same as the class (program lifetime) |

```java
class Employee {
    static int count;           // static, lives for the program's duration
    String name;                // instance, lives with the object

    void greet() {
        int tmp;                // local, lives only in greet()
        // System.out.println(tmp); // compile error: local read before assignment
    }
}
```

> **Note**: instance and static variables auto-initialize to defaults. Local variables do not; the compiler errors if you read an uninitialized local.

> **Reference**: instance and static variables are covered in [OOP Foundations](<12. OOP Foundations.md>).

### `final` Keyword

Makes a variable's reference unchangeable after its first assignment.

```java
final int MAX = 100;
MAX = 200;                 // compile error, can't reassign

final List<String> items = new ArrayList<>();
items.add("apple");        // OK, mutating the object, not the reference
items = new ArrayList<>(); // compile error, can't reassign
```

- A `final` variable must be assigned exactly once (at declaration, or in the constructor for instance fields).
- `final` prevents reference reassignment, not object mutation.

> **Reference**: `final` in depth, see [Encapsulation & Access Control](<13. Encapsulation & Access Control.md#final Keyword>).

### `var` Keyword (Java 10+)

Local variable type inference: the compiler deduces the type from the initializer. Restricted to local variables that have an initializer, never fields or method parameters.

```java
// Valid
var name = "Alice";                 // String
var count = 42;                     // int
var list = new ArrayList<String>(); // ArrayList<String>
var arr  = new int[]{1, 2};         // arrays need explicit new T[]{...}
// (var a, var b) -> a + b          // also allowed in lambda params (Java 11+)

// Invalid (compile errors)
var x;                              // no initializer to infer from
var val  = null;                    // null has no type
var nums = {1, 2};                  // array shortcut needs an explicit type
public var field = "Bob";           // fields not allowed
public void greet(var x) {}         // parameters not allowed
```

> **Note**: use `var` only when the inferred type is obvious (`var users = new ArrayList<User>();`); avoid it when it hides the type (`var result = service.fetch();`).

## Primitive Data Types

Java has exactly 8 primitive types:

| Type      | Size                                      | Range                                  | Default    |
| :-------- | :---------------------------------------- | :------------------------------------- | :--------- |
| `byte`    | 1 byte (8 bits)                           | -128 to 127                            | `0`        |
| `short`   | 2 bytes (16 bits)                         | -32,768 to 32,767                      | `0`        |
| `int`     | 4 bytes (32 bits)                         | -2,147,483,648 to 2,147,483,647        | `0`        |
| `long`    | 8 bytes (64 bits)                         | -9.2 × 10¹⁸ to 9.2 × 10¹⁸              | `0L`       |
| `float`   | 4 bytes (32 bits)                         | IEEE 754 single precision (~7 digits)  | `0.0f`     |
| `double`  | 8 bytes (64 bits)                         | IEEE 754 double precision (~15 digits) | `0.0`      |
| `char`    | 2 bytes (16 bits)                         | 0 to 65,535 (Unicode, unsigned)        | `'\u0000'` |
| `boolean` | implementation-defined (typically 1 byte) | `true` / `false`                       | `false`    |

All numeric types are signed; `char` is uniquely **unsigned** (a 16-bit Unicode code unit) yet supports standard arithmetic.

```java
int i = 100;                  // int: default whole-number type
long l = 10_000_000_000L;     // long: 'L' suffix needed to avoid int overflow (always uppercase)
float f = 3.14f;              // float: 'f' suffix required; bare decimals are double
double d = 3.14;              // double: default floating-point type (optional 'd')
char grade = 'A';             // char: unsigned 16-bit Unicode literal
boolean active = true;        // boolean: true or false only
```

## Literals

Literals are raw values written directly in source.

```java
// 1. Integer literals (default: int)
int dec = 42;
int hex = 0x2A;            // base 16 (prefix 0x or 0X)
int bin = 0b101010;        // base 2 (prefix 0b or 0B, Java 7+)
int oct = 052;             // base 8 (leading 0, avoid: easy to misread)
long val = 100_000_000L;   // 'L' suffix for long; underscores aid readability

// 2. Floating-point literals (default: double)
double d = 3.14;
float f = 5.6f;            // 'f'/'F' suffix mandatory for float

// 3. Character literals (single-quoted)
char letter = 'A';
char tab = '\t';           // escape sequence
char unicode = '\u0041';   // Unicode escape for 'A' (U+0041)

// 4. String, boolean & null literals
String s = "hello";        // double-quoted
boolean active = true;     // true or false
Object obj = null;         // null literal
```

### Escape Sequences

| Escape   | Meaning                          |
| :------- | :------------------------------- |
| `\n`     | Newline                          |
| `\t`     | Tab                              |
| `\r`     | Carriage return                  |
| `\\`     | Literal backslash                |
| `\'`     | Literal single quote             |
| `\"`     | Literal double quote             |
| `\b`     | Backspace                        |
| `\f`     | Form feed                        |
| `\0`     | Null character                   |
| `\uXXXX` | Unicode character (4 hex digits) |

### Underscore Separators (Java 7+)

Underscores may sit between digits to make long numbers readable:

- **Rule**: allowed only between digits (`1_000` is valid; `_1000`, `1000_`, and `1_.5` are compile errors).
- **String parsing**: underscores are not stripped from input strings, so `Integer.parseInt("1_000")` throws `NumberFormatException`.

## Wrapper Classes

Each primitive has an object wrapper class:

| Primitive | Wrapper     |
| :-------- | :---------- |
| `byte`    | `Byte`      |
| `short`   | `Short`     |
| `int`     | `Integer`   |
| `long`    | `Long`      |
| `float`   | `Float`     |
| `double`  | `Double`    |
| `char`    | `Character` |
| `boolean` | `Boolean`   |

Wrappers are needed when:

- Collections cannot store primitives, so `List<int>` is illegal and you use `List<Integer>`.
- A value must be nullable, since primitives cannot be `null` but `Integer x = null` is fine.
- A generic type argument is required, as `<T>` binds only to reference types.

```java
// Boxing (primitive → wrapper)
int primitive = 42;
Integer wrapped = primitive;      // auto-boxing (Java 5+)

// Unboxing (wrapper → primitive)
Integer wrapped2 = 42;
int primitive2 = wrapped2;        // auto-unboxing

// Collections must use wrappers
List<Integer> nums = new ArrayList<>();
nums.add(42);                     // auto-box: int → Integer
int first = nums.get(0);          // auto-unbox: Integer → int
```

> **Note**: autoboxing caches `Integer` (and `Byte`/`Short`/`Long`/`Character`) objects for the range −128 to 127, so `==` can appear correct for small values but fails above 127. Compare wrapper values with `.equals()`, never `==`. Unboxing a `null` wrapper, e.g. `int x = map.get(missing)`, throws `NullPointerException`.

## Non-Primitive Data Types

Anything that is not a primitive is a **reference type**: the variable holds a reference to an object on the heap.

| Type           | Description                                                           | Chapter                                                          |
| :------------- | :-------------------------------------------------------------------- | :--------------------------------------------------------------- |
| **String**     | Immutable character sequence                                          | [Strings](<09. Strings.md>)                                      |
| **Arrays**     | Fixed-size ordered collection of same-type values                     | [Arrays](<08. Arrays.md>)                                        |
| **Classes**    | User-defined types (blueprints for objects)                           | [OOP Foundations](<12. OOP Foundations.md>)                      |
| **Interfaces** | Contract of behaviors (no state; implemented by classes)              | [Abstraction & Interfaces](<16. Abstraction & Interfaces.md>)    |
| **Enums**      | Fixed set of named constants (can carry state and behavior)           | [Enums](<20. Enums.md>)                                          |

```java
String name = "Alice";                // String
int[] numbers = {1, 2, 3};            // array
Employee emp = new Employee("Alice"); // class instance
```

## Q&A

1. **How does local-variable initialization differ from instance/static?**
Instance and static variables are auto-initialized to defaults (`0`, `null`, `false`) when the object or class is loaded. Local variables live on the stack and must be explicitly assigned before they are read; the compiler enforces this to prevent reading indeterminate data.

2. **Why do numeric literals need suffixes like `L` or `f`?**
By default whole-number literals are `int` and decimal literals are `double`, so a suffix is required when the target type differs. Without `L`, `10_000_000_000` is treated as an `int` and overflows; without `f`, a `double` literal won't assign to a `float`.

3. **Does a `final` reference make the object immutable?**
No. `final` only stops the variable from being reassigned to another object; the object's own state can still change (you can `add` to a `final List`). True immutability requires the class to be designed that way, as with `String` or `Integer`.

4. **What does `var list = new ArrayList<>()` infer, and why is that a problem?**
With `var` there is no target type for the diamond to infer from, so it defaults to `ArrayList<Object>`, silently discarding element-type safety. Supply the type in the diamond (`new ArrayList<String>()`) or use an explicit declaration.

5. **Why are two equal `Integer` values sometimes `==` and sometimes not?**
Autoboxing reuses cached `Integer` objects for −128 to 127, so `Integer a = 100, b = 100; a == b` is `true` (same cached instance) but at `200` it is `false` (distinct objects). `==` compares references for wrappers, so compare values with `.equals()`.

## Quick Revision Cheat Sheet

| Concept                     | Remember                                                                       |
| :-------------------------- | :----------------------------------------------------------------------------- |
| Local var default           | None, must initialize before use                                               |
| Instance/static var default | `0`, `0.0`, `false`, `null` (auto-initialized)                                 |
| `final`                     | Reassignment forbidden; object mutation still allowed                          |
| `var` (Java 10+)            | Local-only type inference; no initializer = error; diamond infers `Object`     |
| Primitives                  | 8 types: `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean`  |
| Literal defaults            | Whole numbers `int` (suffix `L` for long); decimals `double` (suffix `f` for float) |
| `char`                      | 16-bit unsigned Unicode                                                        |
| Integer formats             | Decimal, `0x` hex, `0b` binary, `0` octal                                      |
| Underscore separators       | `1_000_000`, between digits only (Java 7+)                                     |
| Wrappers                    | `Integer`, `Long`, etc.; needed for collections, nullability, generics         |
| Wrapper equality            | `==` is identity; use `.equals()`; `Integer` caches −128…127; null unbox → NPE  |
| Reference types             | String, arrays, classes, interfaces, enums; variables hold references          |
