Functional programming (Java 8+) lets you treat behavior as data: pass a function as an argument, return it, or store it in a variable. A lambda supplies that function inline, replacing a single-method anonymous class.

```java
// Before Java 8: anonymous class
Collections.sort(list, new Comparator<String>() {
    public int compare(String a, String b) { return a.length() - b.length(); }
});

// Java 8+: lambda
list.sort((a, b) -> a.length() - b.length());
```

## Functional Interfaces

A functional interface declares exactly one abstract method, the SAM (single abstract method); a lambda or method reference becomes that method's implementation. `default` and `static` methods, and `public` methods inherited from `Object` (like `equals`), do not count toward the one.

```java
@FunctionalInterface       // optional; compiler enforces the SAM rule
interface Greeting {
    void sayHello(String name);          // the one abstract method
    default void hi() {}                 // does not count
    static Greeting noop() { return n -> {}; }  // does not count
    boolean equals(Object o);            // Object method, does not count
}

Greeting g = name -> System.out.println("Hello " + name);
g.sayHello("Alice");   // Hello Alice
```

### Built-in Interfaces (java.util.function)

These cover most cases, so a custom functional interface is rarely needed.

| Interface           | Method      | In   | Out     | Use                          |
| ------------------- | ----------- | ---- | ------- | ---------------------------- |
| `Predicate<T>`      | `test(T)`   | T    | boolean | Filtering, conditions        |
| `Function<T,R>`     | `apply(T)`  | T    | R       | Transform / map              |
| `Consumer<T>`       | `accept(T)` | T    | void    | Side effect (forEach)        |
| `Supplier<T>`       | `get()`     | none | T       | Provide / generate a value   |
| `UnaryOperator<T>`  | `apply(T)`  | T    | T       | Transform, same type         |
| `BinaryOperator<T>` | `apply(T,T)`| T, T | T       | Combine two values (reduce)  |
| `BiFunction<T,U,R>` | `apply(T,U)`| T, U | R       | Two inputs, one output       |
| `BiConsumer<T,U>`   | `accept(T,U)`| T, U| void    | Action on two (Map.forEach)  |

```java
Predicate<Integer> isEven = n -> n % 2 == 0;          isEven.test(4);          // true
Function<String, Integer> len = String::length;       len.apply("hello");      // 5
Consumer<String> print = System.out::println;         print.accept("hi");
Supplier<Double> rnd = Math::random;                  rnd.get();
UnaryOperator<String> up = String::toUpperCase;       up.apply("hi");          // "HI"
BinaryOperator<Integer> sum = (a, b) -> a + b;        sum.apply(3, 4);         // 7
BiFunction<String, String, Integer> both = (a, b) -> a.length() + b.length();  // 2-in, 1-out
```

### Chaining

```java
Predicate<Integer> isEven = n -> n % 2 == 0, isPos = n -> n > 0;
isEven.and(isPos);   isEven.or(isPos);   isEven.negate();

Function<String, String> trim = String::trim, up = String::toUpperCase;
trim.andThen(up).apply("  hi  ");   // trim then up  -> "HI"
trim.compose(up).apply("  hi  ");   // up then trim  -> reverse order

Consumer<String> log = s -> System.err.println("LOG: " + s);
print.andThen(log);   // runs both, in order

// Comparator chaining
list.sort(Comparator.comparing(String::length).reversed());                 // longest first
list.sort(Comparator.comparing(String::length).thenComparing(String::compareTo)); // length, then lexical
```

### Primitive Specializations

Primitive variants (`IntPredicate`, `IntUnaryOperator`, `ToIntFunction<T>`, `IntSupplier`, and the `Long`/`Double` equivalents) avoid autoboxing on hot paths.

```java
IntPredicate isEven = n -> n % 2 == 0;      isEven.test(4);        // no Integer boxing
IntUnaryOperator twice = n -> n * 2;        twice.applyAsInt(5);   // 10
IntSupplier roll = () -> (int) (Math.random() * 6);  roll.getAsInt();
```

## Lambda Expressions

A lambda is an anonymous function that implements the single abstract method of a functional interface.

```java
(parameters) -> { body; return value; }     // full form

(a, b) -> a + b                 // single expression: no braces, no return
a -> a * 2                      // single param: parentheses optional
() -> System.out.println("hi")  // no params: empty parentheses required

(a, b) -> {                     // multiple statements: braces and return required
    int s = a + b;
    return s;
}
```

Parameter types are optional (the compiler infers them from the target). A lambda is only assignable to a functional-interface type, has no name, access modifier, or declared return type, and shares `this` with the enclosing instance (unlike an anonymous class). It reads instance and static fields freely but can capture a local only if it is effectively final.

## Method References

Shorthand for a lambda that only forwards its arguments to an existing method.

| Kind                      | Syntax                  | Equivalent lambda                  |
| ------------------------- | ----------------------- | ---------------------------------- |
| Static                    | `ClassName::staticFn`   | `(args) -> ClassName.staticFn(args)`|
| Bound instance (object)   | `object::method`        | `(args) -> object.method(args)`     |
| Unbound instance (type)   | `ClassName::method`     | `(obj, args) -> obj.method(args)`   |
| Constructor               | `ClassName::new`        | `(args) -> new ClassName(args)`     |

```java
Function<String, Integer> parse = Integer::parseInt;   // s -> Integer.parseInt(s)
Function<String, String> greet = "Hello "::concat;     // bound: s -> "Hello ".concat(s)
Function<String, String> up = String::toUpperCase;     // unbound: s -> s.toUpperCase()

// Unbound is the subtle one: the first parameter becomes the receiver
BiPredicate<String, String> starts = String::startsWith;   // (a, b) -> a.startsWith(b)

Supplier<List<String>> newList = ArrayList::new;       // () -> new ArrayList<>()
Function<String, StringBuilder> sb = StringBuilder::new; // s -> new StringBuilder(s)
```

Use a method reference when the lambda merely calls a method; keep a lambda when you add logic (`s -> System.out.println("Item: " + s)` cannot be a reference).

## Effectively Final Variables

A lambda may capture a local variable only if it is effectively final: assigned once and never reassigned. The capture is by value, so a variable that could change afterward would leave the lambda's copy out of sync, hence the restriction. Instance and static fields are exempt; they are captured by reference.

```java
int factor = 3;                                  // effectively final
Function<Integer, Integer> scale = n -> n * factor;   // OK

int counter = 0;
// counter++;                                    // any reassignment breaks the capture below
// Consumer<String> c = s -> counter++;          // compile error: not effectively final
```

To mutate captured state, hold it behind a final reference:

```java
int[] count = {0};                               // the array is final, its contents are not
list.forEach(s -> count[0]++);

AtomicInteger atomic = new AtomicInteger();      // same idea, and thread-safe
list.forEach(s -> atomic.incrementAndGet());
```

## Q&A

1. **How does `this` differ in a lambda versus an anonymous inner class?**
A lambda creates no scope of its own: `this` is the enclosing instance, and the lambda holds no state. An anonymous class is a new scope whose `this` is the anonymous instance, and it can declare its own fields.

```java
class Outer {
    Runnable lambda = () -> System.out.println(this);   // the Outer instance
    Runnable anon = new Runnable() {
        int count = 0;                                  // anon can hold state
        public void run() { System.out.println(this); } // the anon instance
    };
}
```

2. **Can a functional interface extend another interface?**
Yes, as long as the combined hierarchy still resolves to exactly one abstract method.

3. **Why does `String::toUpperCase` look static but isn't?**
It is an unbound instance reference: the first argument the functional interface supplies becomes the receiver. So `Function<String,String> f = String::toUpperCase` is `s -> s.toUpperCase()`.

4. **Why can't a lambda be assigned to `var`?**
A lambda has no type of its own; it takes the type of its functional-interface target. With `var` there is no target to infer from, so it fails to compile.

```java
var fn = (a, b) -> a + b;                          // compile error: no target type
Comparator<String> fn = (a, b) -> a.compareTo(b);  // OK: target is explicit
```

5. **Why can't a standard lambda throw a checked exception?**
The lambda implements the interface's abstract method, and built-in interfaces like `Function` and `Predicate` declare no `throws`. To call checked-throwing code, catch and wrap it in an unchecked exception, or define a functional interface that declares the checked type.

## Quick Revision Cheat Sheet

| Concept              | Remember                                                              |
| -------------------- | -------------------------------------------------------------------- |
| Functional interface | Exactly one abstract method; `Object`/`default`/`static` don't count |
| @FunctionalInterface | Optional; compiler enforces the SAM rule                             |
| Lambda target type   | No type of its own; needs a functional-interface target (no bare `var`) |
| this in lambda       | Enclosing instance, not a new scope (unlike an anonymous class)      |
| Effectively final    | Captured locals must never be reassigned (captured by value)        |
| Method reference     | Four kinds: static, bound instance, unbound instance, constructor   |
| Unbound instance ref | `String::toUpperCase` makes the first argument the receiver         |
| Chaining             | Predicate and/or/negate; Function andThen/compose; Consumer andThen |
| Primitive variants   | `IntPredicate`, `ToIntFunction`, etc. avoid autoboxing              |
