## Reading Input

Java reads console input through `Scanner` for parsed tokens and `Console` for secure, unechoed input.

### `Scanner` Class

The standard utility for reading and parsing console input. `Scanner` wraps `System.in` (the JVM's standard input, a `java.io.InputStream` byte stream) and parses primitives and strings using regular expressions.

```java
Scanner sc = new Scanner(System.in);
System.out.print("Enter name: ");
String name = sc.nextLine();
System.out.print("Enter age: ");
int age = sc.nextInt();
sc.close();
```

Common methods:

```java
sc.next()          // next whitespace-delimited token (word)
sc.nextLine()      // rest of the current line (as String)
sc.nextInt()       // next token parsed as int
sc.nextLong()      // next token parsed as long
sc.nextDouble()    // next token parsed as double
sc.nextBoolean()   // next token parsed as boolean

sc.hasNext()       // is there more input? (used in loops)
sc.hasNextInt()    // is the next token an int?
sc.hasNextLine()   // is there another line?
```

The leftover-newline problem: methods like `nextInt()` and `nextDouble()` consume only the value and leave the trailing newline (`\n`) in the buffer, so a following `nextLine()` reads that empty newline instead of waiting for input.

```java
Scanner sc = new Scanner(System.in);
System.out.print("Age: ");
int age = sc.nextInt();           // reads "30", leaves \n in the buffer
System.out.print("Name: ");
String name = sc.nextLine();      // reads the leftover \n, skips input!
```

Fix it by consuming the stray newline explicitly:

```java
int age = sc.nextInt();
sc.nextLine();                    // consume the leftover newline
String name = sc.nextLine();      // now reads the name correctly
```

Or avoid the buffer issue entirely by reading every input as a line and parsing it yourself:

```java
int age = Integer.parseInt(sc.nextLine());      // throws NumberFormatException if invalid
double gpa = Double.parseDouble(sc.nextLine());
boolean active = Boolean.parseBoolean(sc.nextLine()); // "true" (case-insensitive) → true; else false
```

> **Note**: `parseInt`/`parseDouble` are static factory methods on the wrapper classes; you call them without an instance to convert a `String` to a primitive.

> **Note**: for bulk or performance-sensitive input, `BufferedReader br = new BufferedReader(new InputStreamReader(System.in));` with `br.readLine()` is faster than `Scanner` (no regex parsing) but returns raw `String`s you must parse yourself.

### `Console` Class

Provides a `java.io.Console` that can read input without echoing it to the screen.

```java
Console console = System.console();
if (console != null) {
    char[] password = console.readPassword("Password: ");
    // use password...
    Arrays.fill(password, ' ');           // clear from memory once done
} else {
    System.out.println("No console available"); // IDEs, pipes, redirects
}
```

- Returns a `char[]`, not a `String`, so the password can be wiped immediately (immutable Strings linger until garbage collection).
- Returns `null` when stdin is redirected or run inside an IDE, so always null-check it.

## Printing Output

Console output goes through `PrintStream` (`System.out` and `System.err`), with `printf` for formatted text.

### `PrintStream` Class

`System.out` and `System.err` are both `java.io.PrintStream` instances:

- `System.out` is standard output (stdout), line-buffered to the terminal.
- `System.err` is standard error (stderr), unbuffered for immediate display.

### `print()` and `println()`

```java
System.out.println("Hello");            // prints "Hello\n"
System.out.print("Hello");              // prints "Hello" (no newline)
System.out.println();                   // prints a newline
System.out.println(42);                 // converts via String.valueOf
System.out.println(new int[]{1, 2, 3}); // prints a class descriptor; use Arrays.toString()
```

### Formatted Output with `printf()`

`printf()` (and its alias `format()`) formats output using placeholders:

```java
System.out.printf("Name: %s, Age: %d%n", "Alice", 30); // "Name: Alice, Age: 30\n"

// Common specifiers (use %n for a platform-portable newline)
System.out.printf("%d%n", 42);            // 42          (integer)
System.out.printf("%,d%n", 1_000_000);    // 1,000,000   (comma grouping)
System.out.printf("%05d%n", 42);          // 00042       (zero-padded, width 5)
System.out.printf("%.2f%n", 3.14159);     // 3.14        (2 decimal places)
System.out.printf("%-10s|%n", "hi");      // hi        | (left-aligned, width 10)
System.out.printf("%%%n");                // %           (literal percent)
```

Prefer `%n` over `\n` in `printf`: it resolves to the system line separator (`\r\n` on Windows, `\n` on macOS/Linux).

## Q&A

1. **How do you handle the leftover newline when mixing numeric and text input?**
`nextInt()` and friends leave the trailing `\n` in the buffer, so the next `nextLine()` reads that empty newline and skips your input. Consume it with an extra `sc.nextLine()`, or read everything with `nextLine()` and parse the values yourself.

2. **Why null-check `System.console()` before using it?**
`System.console()` returns `null` whenever the JVM is not attached to an interactive terminal, which happens inside IDEs, background processes, and redirected streams.

3. **Why does `Console.readPassword()` return `char[]` instead of `String`?**
Strings are immutable and stay in memory until garbage collection, leaving a password exposed to heap dumps. A `char[]` is mutable, so you can zero it out the moment you're done.

## Quick Revision Cheat Sheet

| Concept            | Remember                                                                           |
| :----------------- | :--------------------------------------------------------------------------------- |
| Scanner newline    | `nextInt()` leaves `\n` in the buffer; consume it before `nextLine()`              |
| Scanner vs BufferedReader | Scanner parses tokens (convenient); BufferedReader reads raw lines (faster) |
| `System.console()` | Returns `null` in IDEs, redirects, background processes; always null-check         |
| Secure input       | `readPassword()` returns a mutable `char[]` for immediate memory erasure            |
| Buffering          | `System.out` is line-buffered; `System.err` is unbuffered                          |
| `printf`           | Use `%n` (not `\n`) for portable newlines; `%s %d %.2f %,d %05d %-10s`             |
