## Reading Input

### `Scanner` Class

The standard utility to read and parse console input. The `Scanner` wraps `System.in` (the JVM's standard input stream, represented as a `java.io.InputStream` byte stream) to parse primitive types and strings using regular expressions.

```java
import java.util.Scanner;

public class ScannerDemo {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter name: ");
        String name = sc.nextLine();
        System.out.print("Enter age: ");
        int age = sc.nextInt();
        sc.close();
    }
}
```

**Common Scanner Methods**:

```java
sc.next()          // next whitespace-delimited token (word)
sc.nextLine()      // rest of the current line (as String)
sc.nextInt()       // next token parsed as int
sc.nextLong()      // next token parsed as long
sc.nextDouble()    // next token parsed as double
sc.nextBoolean()   // next token parsed as boolean
sc.nextByte()      // byte
sc.nextShort()     // short
sc.nextFloat()     // float

sc.hasNext()       // is there more input? (used in loops)
sc.hasNextInt()    // is the next token an int?
sc.hasNextLine()   // is there another line?
```

### Input Buffer Behavior
Methods like `nextInt()` and `nextDouble()` only consume the input value, leaving the trailing newline character (`\n`) in the buffer. A subsequent `nextLine()` will immediately read this leftover newline instead of waiting for new input.

```java
Scanner sc = new Scanner(System.in);
System.out.print("Age: ");
int age = sc.nextInt();           // Reads "30" — leaves \n in buffer
System.out.print("Name: ");
String name = sc.nextLine();      // Reads the leftover \n — skips input!
```

To resolve this, consume the stray newline explicitly:

```java
int age = sc.nextInt();
sc.nextLine();                    // Consumes the leftover newline
String name = sc.nextLine();      // Correctly reads the name input
```

Alternatively, read everything as strings and parse manually:

```java
int age = Integer.parseInt(sc.nextLine().trim());
String name = sc.nextLine();
```

**Numeric Parsing**:
To avoid buffer issues, read inputs as strings via `nextLine()` and parse them manually using static wrapper methods:

```java
System.out.print("Enter age: ");
int age = Integer.parseInt(sc.nextLine());      // Parses as int (throws NumberFormatException if invalid)

System.out.print("Enter GPA: ");
double gpa = Double.parseDouble(sc.nextLine()); // Parses as double

System.out.print("Is enrolled: ");
boolean active = Boolean.parseBoolean(sc.nextLine()); // "true" (case-insensitive) -> true; all else -> false
```

These utility methods are static because primitive types cannot have methods.

### `Console` Class

Retrieves a `java.io.Console` that reads input without echoing characters on the screen.

```java
Console console = System.console();
if (console != null) {
    char[] password = console.readPassword("Password: ");
    // use password...
    Arrays.fill(password, ' ');           // Clear from memory for security
} else {
    // Fallback to Scanner if console is null (e.g. inside IDEs or pipes)
    System.out.println("No console available");
}
```

- Returns `char[]` instead of `String` so the password can be cleared from memory immediately (immutable Strings linger until garbage collection).
- Returns `null` if stdin is redirected or run inside an IDE; always perform a null check.

## Printing Output

### `PrintStream` Class

`System.out` and `System.err` are `java.io.PrintStream` instances:
- `System.out` is the standard output stream (stdout), which is line-buffered to the terminal.
- `System.err` is the standard error stream (stderr), which is unbuffered for immediate display.

### `println()`, `print()`, `printf()`

```java
System.out.println("Hello");       // Prints "Hello\n"
System.out.print("Hello");         // Prints "Hello" (no newline)
System.out.println();              // Prints a newline
System.out.println(42);            // Converts via String.valueOf
System.out.println(new int[]{1,2,3}); // Prints class name and hash code; use Arrays.toString()

System.out.printf("Name: %s, Age: %d%n", "Alice", 30); // Prints "Name: Alice, Age: 30\n"
```

Prefer `%n` over `\n` in `printf` for platform portability; `%n` automatically resolves to the system's local line separator (`\r\n` on Windows, `\n` on macOS/Linux).

### Format specifiers

```java
System.out.printf("%d", 42);            // 42                (integer)
System.out.printf("%,d", 1_000_000);    // 1,000,000         (grouping)
System.out.printf("%5d", 42);           //    42             (width 5, right-aligned)
System.out.printf("%-5d|", 42);         // 42   |            (left-aligned)
System.out.printf("%05d", 42);          // 00042             (zero-padded)

System.out.printf("%f", 3.14159);       // 3.141590
System.out.printf("%.2f", 3.14159);     // 3.14              (2 decimal places)
System.out.printf("%10.2f", 3.14);      //       3.14        (width 10, 2 decimals)

System.out.printf("%s", "hello");       // hello
System.out.printf("%10s", "hi");        //         hi        (right-aligned in width 10)
System.out.printf("%-10s|", "hi");      // hi        |       (left-aligned)

System.out.printf("%c", 'A');           // A
System.out.printf("%b", true);          // true
System.out.printf("%%");                // %                 (literal percent sign)
```

> **Cross-reference**: For advanced number parsing, decimal formatting, and locales, see [Numbers — Parsing and Formatting](<10. Numbers, Math & Random.md#Parsing and formatting>).

## Q&A

1. **What is the `nextLine()` behavior when using `Scanner`, and how do you resolve it?**
Methods like `nextInt()` leave the trailing newline character (`\n`) in the input buffer. A subsequent `nextLine()` reads this empty newline immediately, skipping your intended input. To fix this, consume the newline with an extra `sc.nextLine()`, or read everything via `nextLine()` and parse manually.

2. **Why should you null-check `System.console()` before using it?**
`System.console()` returns `null` if the JVM is not attached to an interactive terminal. This occurs when running code inside IDEs, background processes, or redirected output streams.

3. **Why does `Console.readPassword()` return a `char[]` instead of a `String`?**
Strings are immutable and remain in memory until garbage collection, leaving passwords vulnerable to memory dumps. A `char[]` is mutable, allowing you to zero out the characters immediately after use.

4. **What happens when you call `System.out.println(new int[]{1, 2, 3})`, and what is the correct way to print it?**
It will print the object's class name and hash code (e.g., `[I@1b6d3586`) rather than the array elements. Use `Arrays.toString(arr)` for 1D arrays, and `Arrays.deepToString(arr)` for multi-dimensional arrays.

5. **When using `printf`, why is `%n` preferred over `\n`?**
The `%n` specifier represents the platform-specific line ending (`\r\n` on Windows, `\n` on macOS/Linux). Using `%n` ensures your output matches the OS line-ending standards, avoiding compatibility issues with other tools.

## Quick Revision Cheat Sheet

| Concept            | Remember                                                                    |
| :----------------- | :-------------------------------------------------------------------------- |
| `Scanner`          | Wraps `System.in` for typed parsing (`nextInt()`, `nextLine()`, etc.)       |
| `System.in`        | The standard `InputStream` byte stream provided by the JVM                  |
| `nextLine()`       | Consumes trailing newlines left in buffer by numeric parsing methods        |
| Parse methods      | Static methods on wrapper classes (e.g., `Integer.parseInt()`)               |
| `Console`          | Secure `readPassword()` method returning mutable `char[]`                   |
| `System.console()` | Returns `null` inside IDEs or background processes; always null-check       |
| `System.out`       | A line-buffered `PrintStream` for standard output                           |
| `%n` vs `\n`       | Use `%n` for platform-independent newline characters in `printf`            |
| Print an array     | Use `Arrays.toString()` or `Arrays.deepToString()`, not `System.out.println` |
| `System.err`       | An unbuffered `PrintStream` for immediate error display                     |
