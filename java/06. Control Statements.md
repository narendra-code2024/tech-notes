## Conditional Statements

Statements that select a branch of execution based on a condition.

### `if` / `if-else` / `else-if`

```java
if (condition) {
    // runs if condition is true
}

if (score >= 90) {
    grade = "A";
} else if (score >= 80) {
    grade = "B";
} else if (score >= 70) {
    grade = "C";
} else {
    grade = "F";
}
```

- The condition must resolve to a `boolean` (or a `Boolean`, which is auto-unboxed, so a `null` `Boolean` throws `NullPointerException`).
- `else if` is not a keyword; it is just an `else` whose body is another `if`.
- Braces are optional for single-line bodies but strongly recommended to avoid **dangling-else** bugs.

### `switch` Statement

The classic `switch` statement uses `break` to prevent fall-through.

```java
switch (day) {
    case 1:
        System.out.println("Monday");
        break; // correct: exits the switch
    case 2:
        System.out.println("Tuesday");
        break;

    case 3:
    case 4: // intentional grouping (Wednesday–Thursday)
        System.out.println("Midweek");
        break;

    case 5:
        System.out.println("Friday");
        // missing break! accidental fall-through bug
    case 6:
    case 7: // intentional grouping (Saturday–Sunday)
        System.out.println("Weekend"); // if day is 5, prints BOTH "Friday" and "Weekend"
        break;

    default:
        System.out.println("Unknown Day");
}
```

- Supports `byte`, `short`, `int`, `char` (and their wrappers), `String` (Java 7+), and `enum`; not `long`, floating-point, or `boolean`.
- Omitting `break` falls through into the next case. Useful for multi-value grouping but a common source of bugs; `switch` expressions remove this behavior.

### `switch` Expressions (Java 14+)

A `switch` expression returns a value and removes fall-through using arrow syntax.

```java
String name = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3, 4, 5 -> "Midweek";               // multi-value case
    default -> {
        log.warn("Unknown day: " + day);
        yield "Unknown";                      // yield returns a value from a block arm
    }
};
```

These improve on the statement form in several ways:

- Arrow syntax (`->`) removes fall-through, so only the matching branch runs.
- Case constants can be comma-separated (`case 1, 2, 3 ->`).
- It evaluates to a value, so it can be assigned or returned directly.
- The compiler enforces exhaustiveness, so an `enum` switch covering all constants needs no `default`, and any other switch does.

Arms can be single expressions or `{ ... }` blocks; a block arm must use `yield` to return a value (omitting it is a compile error).

### `switch` Statement vs `switch` Expression

| Feature          | `switch` Statement           | `switch` Expression (Java 14+) |
| :--------------- | :--------------------------- | :----------------------------- |
| Syntax           | `case X: ... break;`         | `case X -> ...`                |
| Fall-through     | Supported (requires `break`) | Not supported                  |
| Returns value    | No (statement)               | Yes (expression)               |
| Exhaustiveness   | Not enforced                 | Enforced                       |
| Multi-value case | Multiple labels              | Comma-separated list           |
| Primary use      | Side effects                 | Value production               |

## Iterative Statements

Loops that repeat a body while a condition holds.

### `for`

Runs with three optional parts: initialization, condition, and update. `for (;;) { }` is a legal infinite loop.

```java
for (int i = 0; i < 10; i++) {
    System.out.println(i);
}
```

### `while`

Checks the condition before each iteration, so it runs zero or more times. Ideal when the iteration count is unknown upfront.

```java
while (!queue.isEmpty()) {
    process(queue.poll());
}
```

### `do-while`

Runs the body once before checking the condition, so it runs at least once. Useful for retry logic or input validation.

```java
int attempts = 0;
do {
    System.out.println("Try " + attempts);
    attempts++;
} while (attempts < 3);
```

### Enhanced `for-each` (Java 5+)

Iterates over arrays or `Iterable` collections without manual index management.

```java
List<String> names = List.of("Alice", "Bob", "Charlie");
for (String name : names) {
    System.out.println(name);
}

// Maps aren't Iterable, iterate entrySet()
Map<String, Integer> scores = Map.of("Alice", 90, "Bob", 85);
for (Map.Entry<String, Integer> entry : scores.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}
```

Use a classic `for` instead of `for-each` when you need to:

- Modify the collection during iteration (otherwise it throws `ConcurrentModificationException`).
- Access the current element's index.
- Iterate in reverse.
- Iterate multiple collections in parallel.

## Branching Statements

Statements that redirect control flow out of, or within, a loop or method.

### `break`

Exits the nearest enclosing loop or `switch`.

```java
for (int i = 0; i < 100; i++) {
    if (i == 50) break;   // exits the loop
    System.out.println(i);
}
```

### `continue`

Skips the rest of the current iteration and proceeds to the next.

```java
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) continue;    // skip even numbers
    System.out.println(i);       // prints 1, 3, 5, 7, 9
}
```

### `return`

Exits the current method, optionally returning a value.

```java
boolean isValid(String s) {
    if (s == null) return false;
    if (s.isEmpty()) return false;
    return s.length() < 100;
}
```

### Labeled Loops

A label lets `break`/`continue` target an outer loop instead of the nearest one.

```java
search:
for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        if (matrix[i][j] == target) {
            System.out.println("Found at " + i + "," + j);
            break search;   // exits both loops
        }
    }
}
```

> **Note**: labeled loops are generally avoided; prefer extracting the nested loops into a method and using an early `return`.

## Q&A

1. **What is fall-through in a `switch`, and why is it dangerous?**
In a classic `switch`, omitting `break` lets execution continue into the following `case` regardless of its label. It is dangerous because it is usually accidental, producing silent logic bugs.

2. **Why does `if (a = true)` compile, and what is the likely mistake?**
`a = true` assigns `true` to `a` and the assignment itself evaluates to `true`, so the condition always passes. The compiler cannot flag it because, for a `boolean`, the assignment expression is a valid boolean condition (the intended code was `a == true`, or just `a`).

3. **What is `ConcurrentModificationException` and why does it strike in a `for-each`?**
Structurally modifying a collection while iterating it invalidates the backing `Iterator`'s state, which it detects and reports as `ConcurrentModificationException`. To remove during iteration, use an explicit `Iterator` and its `remove()`.

4. **When is `default` mandatory in a `switch` expression?**
Always, except when the expression switches over an `enum` and covers every constant; otherwise `default` is required to guarantee compile-time exhaustiveness.

5. **Why can `for (int i = 0; i <= Integer.MAX_VALUE; i++)` loop forever?**
When `i` is `Integer.MAX_VALUE`, `i++` overflows and wraps to `Integer.MIN_VALUE`, which is still `<=` the limit, so the condition never becomes false.

6. **Can you write `case Color.RED:` in a `switch`?**
No. The compiler already knows the enum type from the selector, so a traditional enum `switch` requires the bare constant (`case RED:`); qualifying it as `case Color.RED:` is a compile error.

## Quick Revision Cheat Sheet

| Concept               | Remember                                                         |
| :-------------------- | :--------------------------------------------------------------- |
| `if / else if / else` | Condition must be `boolean`; braces avoid dangling-else bugs     |
| `switch` statement    | Classic form with fall-through; needs `break` to exit            |
| `switch` expression   | Java 14+; returns a value, arrow syntax, exhaustive              |
| `for`                 | Counter-controlled; checks condition before each run             |
| `while` / `do-while`  | `while` checks first (0+ runs); `do-while` checks last (1+ runs) |
| Enhanced `for-each`   | Simple iteration; cannot structurally modify mid-loop            |
| `break` / `continue`  | `break` exits the loop; `continue` skips to the next iteration   |
| `return`              | Exits the current method, optionally with a value                |
| Labeled loops         | Target an outer loop from a nested one; usually avoidable        |
| `yield`               | Returns a value from a `switch`-expression block arm             |
