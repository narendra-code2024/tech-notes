Arrays are **fixed-size, ordered** containers that store elements of the **same type** in contiguous memory (for primitives) or contiguous references (for objects) using zero-based indexing.

```java
// 1. Declaration & Sized Initialization (defaults to 0)
int[] arr = new int[5];          // Sized initialization; default values [0, 0, 0, 0, 0]
arr[0] = 10;                     // Index-based assignment
arr[4] = 50;
System.out.println(arr.length);  // 5

// 2. Discouraged C-style Declaration
int arrC[];                      // Valid but discouraged

// 3. Array Literals (inferred sizing)
int[] arrLit = {1, 2, 3, 4, 5};            // Shorthand literal syntax
int[] arrExp = new int[]{1, 2, 3, 4, 5};  // Explicit inline initialization
```

**Default Values**:
* Primitive numeric arrays (`int[]`, `double[]`) default to `0` or `0.0`.
* Boolean arrays (`boolean[]`) default to `false`.
* Character arrays (`char[]`) default to `'\u0000'` (null character).
* Object arrays (`String[]` etc.) default to `null`.

> **Note**: Do not confuse array length with string or collection sizes:
> * **Arrays**: `arr.length` (public read-only field; no parentheses)
> * **Strings**: `str.length()` (method; requires parentheses)
> * **Collections**: `list.size()` (method; requires parentheses)

## Multi-Dimensional Arrays

Multi-dimensional arrays are represented in Java as arrays of arrays.

```java
// 2D array
int[][] grid = new int[2][3];  // 2 rows, 3 columns

// 2D array literal (initialized with values)
int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6}
};

matrix[0][0] = 1;      // Row 0, Column 0
matrix.length;         // 2 (number of rows)
matrix[0].length;      // 3 (number of columns in row 0)

// Jagged array (rows with different lengths)
int[][] jagged = new int[3][];
jagged[0] = new int[2];
jagged[1] = new int[4];
jagged[2] = new int[1];

// Iterating using standard nested loops
for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        System.out.print(matrix[i][j] + " ");
    }
}
```

## `Arrays` Utility Class

The `java.util.Arrays` class provides static helper methods for common array operations:

```java
int[] arr = {5, 3, 1, 4, 2};

// Sorting and Searching
Arrays.sort(arr);                         // [1, 2, 3, 4, 5] (in-place ascending)
int index = Arrays.binarySearch(arr, 3);  // Returns index (requires pre-sorted array)

// Reverse sorting (requires reference type wrappers; primitive types are not supported)
Integer[] boxed = {5, 3, 1, 4, 2};
Arrays.sort(boxed, Collections.reverseOrder()); // [5, 4, 3, 2, 1]

// Filling
Arrays.fill(arr, 0);                      // Fills entire array with 0

// Comparison
Arrays.equals(arr1, arr2);                // true if same length and element values
Arrays.deepEquals(arr2D_1, arr2D_2);      // Deep comparison for nested arrays

// Printing (Direct print outputs class name and hash)
System.out.println(Arrays.toString(arr));        // [1, 2, 3, 4, 5] (shallow format)
System.out.println(Arrays.deepToString(matrix));  // [[1, 2], [3, 4]] (nested format)
```

> **Note**: Primitive arrays (`int[]`, etc.) do not support direct reverse sorting because comparators require object references. To reverse-sort a primitive array, either sort it in ascending order and reverse it manually (recommended for performance), or box it to a wrapper object array (e.g., `Integer[]`).

## Array Copying

Java offers three primary ways to duplicate arrays. Note that all three methods perform a **shallow copy** for object arrays (references are copied, sharing underlying objects), while primitives are deep-copied directly.

```java
int[] src = {1, 2, 3, 4, 5};

// 1. Arrays.copyOf (returns a new array)
int[] copy1 = Arrays.copyOf(src, src.length);

// 2. System.arraycopy (native high-speed copy; requires existing destination array)
int[] dest = new int[5];
System.arraycopy(src, 0, dest, 0, src.length);
// Arguments: source, srcPos, dest, destPos, length

// 3. clone() (returns a new array)
int[] copy2 = src.clone();
```

## Varargs (Variable Arguments)

Syntactic sugar that allows passing a variable number of arguments to a method, which Java packages into a standard array internally.

```java
public static int sum(int... numbers) {
    int total = 0;
    for (int n : numbers) total += n;
    return total;
}

sum(1, 2, 3);           // Returns 6
sum(new int[]{1, 2});   // Returns 3 (explicit array is also valid)
sum();                  // Returns 0 (empty call evaluates successfully)
```

**Rules**:
1. Only one varargs parameter is allowed per method signature.
2. The varargs parameter must be the last parameter in the signature.

## Q&A

1. **How are arrays passed to methods in Java, and what happens if you modify the array inside the method?**
Java is strictly pass-by-value. However, because arrays are objects stored on the heap, the value passed to the method is the reference copy to the array. Modifying elements within the method changes the original heap object, but reassigning the parameter reference itself inside the method has no effect outside it.

```java
void modify(int[] arr) {
    arr[0] = 100;   // Modifies the original array
}

void reassign(int[] arr) {
    arr = new int[]{9, 9, 9};   // Does NOT affect the original array
}
```

2. **Why shouldn't you print an array directly using `System.out.println(arr)`?**
Printing an array directly invokes `Object.toString()`, which outputs the class name descriptor and the object's hexadecimal hash code (e.g., `[I@1a2b3c`). To inspect array contents, use `Arrays.toString(arr)` for one-dimensional arrays or `Arrays.deepToString(arr)` for multi-dimensional arrays.

3. **What is Array Covariance, and why is it dangerous?**
Array covariance allows a subclass array to be assigned to a superclass array reference (e.g., `Object[] arr = new String[3]`). This compiles successfully but is highly dangerous because trying to insert an incompatible object type at runtime throws an `ArrayStoreException`.

```java
Object[] arr = new String[3];
arr[0] = 123;   // Compiles, but throws ArrayStoreException at runtime
```

4. **Why does `Arrays.asList(new int[]{1, 2, 3})` not work as expected?**
`Arrays.asList()` accepts a generic varargs block of Objects (`T... a`). Because `int[]` is a single object reference and primitive types cannot serve as generic parameters, the method treats the entire array as a single element, returning a `List<int[]>` of size 1 instead of a `List<Integer>` of size 3.

5. **Does `Arrays.sort()` use the same sorting algorithm for all array types?**
No. For primitive arrays, Java uses **Dual-Pivot Quicksort** (which is not stable). For object arrays, it uses **TimSort** (which is stable, ensuring that equal elements retain their original relative positions).

6. **Can you modify the elements of an array using an enhanced `for-each` loop?**
No. The iteration variable in an enhanced `for` loop holds a local copy of the element value at that index, not a direct reference to the array array slot. Reassigning the loop variable has no effect on the array itself.

```java
for (int n : arr) {
    n = 0;    // Does not change the contents of arr
}
```

## Quick Revision Cheat Sheet

| Concept                | Remember                                                              |
| :--------------------- | :-------------------------------------------------------------------- |
| Fixed size             | Cannot grow/shrink after creation; use `ArrayList` for dynamic sizing |
| `length`               | Public field, not a method; accessed as `arr.length` without `()`     |
| `Arrays.toString()`    | Decodes and prints elements; direct print outputs class name and hash  |
| `Arrays.sort()`        | Primitives: Dual-Pivot Quicksort (unstable); Objects: TimSort (stable) |
| `Arrays.copyOf()`      | Shallow copy; creates a new array but shares object references        |
| Covariant              | `Object[] = new String[]` compiles but risks `ArrayStoreException`    |
| Varargs                | Syntactic sugar (`int... nums`); must be the last method parameter     |
| `Arrays.asList(int[])` | Creates `List<int[]>` instead of `List<Integer>`; box elements first  |
| Jagged arrays          | Multi-dimensional arrays of arrays; rows can have different lengths   |
