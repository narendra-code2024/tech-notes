Arrays are **fixed-size, ordered** containers that store elements of the **same type** in contiguous memory (for primitives) or contiguous references (for objects), using zero-based indexing.

```java
// 1. Declaration & sized initialization (defaults to 0)
int[] arr = new int[5];          // default values [0, 0, 0, 0, 0]
arr[0] = 10;                     // index-based assignment
arr[4] = 50;
System.out.println(arr.length);  // 5

// 2. Discouraged C-style declaration
int arrC[];                      // valid but discouraged

// 3. Array literals
int[] arrLit = {1, 2, 3, 4, 5};           // shorthand literal
int[] arrExp = new int[]{1, 2, 3, 4, 5};  // explicit inline form
```

Elements are auto-initialized to their type's default:

- Numeric primitive arrays (`int[]`, `double[]`) default to `0` / `0.0`.
- `boolean[]` defaults to `false`.
- `char[]` defaults to `'\u0000'` (the null character).
- Object arrays (`String[]`, etc.) default to `null`.

> **Note**: do not confuse array length with string or collection size:
> - **Arrays**: `arr.length` (public read-only field, no parentheses)
> - **Strings**: `str.length()` (method)
> - **Collections**: `list.size()` (method)

## Multi-Dimensional Arrays

In Java a multi-dimensional array is an array of arrays.

```java
// 2D array
int[][] grid = new int[2][3];  // 2 rows, 3 columns

// 2D array literal
int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6}
};

matrix[0][0] = 1;      // row 0, column 0
matrix.length;         // 2 (number of rows)
matrix[0].length;      // 3 (columns in row 0)

// Jagged array (rows of different lengths)
int[][] jagged = new int[3][];
jagged[0] = new int[2];
jagged[1] = new int[4];
jagged[2] = new int[1];

// Standard nested iteration
for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        System.out.print(matrix[i][j] + " ");
    }
}
```

## `Arrays` Utility Class

`java.util.Arrays` provides static helpers for common operations:

```java
int[] arr = {5, 3, 1, 4, 2};

// Sorting and searching
Arrays.sort(arr);                         // [1, 2, 3, 4, 5] (in-place ascending)
int index = Arrays.binarySearch(arr, 3);  // returns index (array must be pre-sorted)

// Reverse sort (reference types only; not primitives)
Integer[] boxed = {5, 3, 1, 4, 2};
Arrays.sort(boxed, Collections.reverseOrder()); // [5, 4, 3, 2, 1]

// Filling and comparison
Arrays.fill(arr, 0);                      // fill entire array with 0
Arrays.equals(arr1, arr2);                // true if same length and element values
Arrays.deepEquals(arr2D_1, arr2D_2);      // deep comparison for nested arrays

// Stream bridge
int total = Arrays.stream(arr).sum();     // IntStream over a primitive array

// Printing (direct print outputs class name + hash)
System.out.println(Arrays.toString(arr));        // [1, 2, 3, 4, 5]
System.out.println(Arrays.deepToString(matrix)); // [[1, 2, 3], [4, 5, 6]]
```

> **Note**: primitive arrays do not support comparator-based reverse sorting (comparators need object references). Either sort ascending and reverse in place (faster), or box to a wrapper array (`Integer[]`) and use `Collections.reverseOrder()`.

## Array Copying

Java offers three main ways to duplicate an array. All three are **shallow** for object arrays (references are copied, the underlying objects shared); primitive values are copied directly.

```java
int[] src = {1, 2, 3, 4, 5};

// 1. Arrays.copyOf (returns a new array)
int[] copy1 = Arrays.copyOf(src, src.length);

// 2. System.arraycopy (native high-speed copy into an existing destination)
int[] dest = new int[5];
System.arraycopy(src, 0, dest, 0, src.length);
// args: source, srcPos, dest, destPos, length

// 3. clone() (returns a new array)
int[] copy2 = src.clone();
```

> **Note**: on a 2D array these copies are shallow at the row level: the new array holds the same `int[]` row references, so mutating a row affects both copies. Clone each row in a loop (`copy[i] = src[i].clone();`) for a deep copy.

## Varargs (Variable Arguments)

Syntactic sugar for passing a variable number of arguments, which Java packs into an array internally.

```java
public static int sum(int... numbers) {
    int total = 0;
    for (int n : numbers) total += n;
    return total;
}

sum(1, 2, 3);           // 6
sum(new int[]{1, 2});   // 3 (an explicit array is also accepted)
sum();                  // 0 (an empty call is valid)
```

Two rules: a method may have only one varargs parameter, and it must be the last parameter in the signature.

## Q&A

1. **How are arrays passed to methods, and what happens if you modify one inside?**
Java is strictly pass-by-value, but for an array the value passed is a copy of the reference to the heap object. Modifying elements through that reference changes the original array; reassigning the parameter to a new array does not affect the caller.

```java
void modify(int[] arr) {
    arr[0] = 100;   // modifies the original array
}
void reassign(int[] arr) {
    arr = new int[]{9, 9, 9};   // does NOT affect the original
}
```

2. **Why shouldn't you print an array with `System.out.println(arr)`?**
It invokes `Object.toString()`, printing the class descriptor and hash code (e.g. `[I@1a2b3c`). Use `Arrays.toString(arr)` for 1D arrays and `Arrays.deepToString(arr)` for multi-dimensional ones.

3. **What is array covariance, and why is it dangerous?**
Covariance lets a subtype array be assigned to a supertype array reference (`Object[] a = new String[3]`). It compiles, but storing an incompatible type throws `ArrayStoreException` at runtime, turning a type error into a runtime failure.

```java
Object[] arr = new String[3];
arr[0] = 123;   // compiles, throws ArrayStoreException at runtime
```

4. **Why does `Arrays.asList(new int[]{1, 2, 3})` misbehave?**
`asList` takes generic varargs `T... a`, and `int` cannot be a type argument, so the whole `int[]` is treated as one element. You get a `List<int[]>` of size 1, not a `List<Integer>` of size 3; use `Integer[]` or an `IntStream` to box first.

5. **Does `Arrays.sort()` use one algorithm for all arrays?**
No. Primitive arrays use Dual-Pivot Quicksort (not stable); object arrays use TimSort (stable), so equal elements keep their original relative order, which matters for multi-key sorts.

6. **Can a `for-each` loop modify array elements?**
No. The loop variable holds a copy of each element's value, not a reference to the slot, so assigning to it changes nothing in the array. Use an indexed `for` to write back.

```java
for (int n : arr) {
    n = 0;    // does not change arr
}
```

## Quick Revision Cheat Sheet

| Concept                | Remember                                                               |
| :--------------------- | :--------------------------------------------------------------------- |
| Fixed size             | Cannot grow/shrink; use `ArrayList` for dynamic sizing                 |
| `length`               | Public field, not a method; `arr.length` with no `()`                  |
| `Arrays.toString()`    | Prints elements; direct print outputs class name + hash                |
| `Arrays.sort()`        | Primitives: Dual-Pivot Quicksort (unstable); objects: TimSort (stable) |
| `Arrays.copyOf()`      | New array but shares object references (shallow; 2D shares rows)       |
| Covariance             | `Object[] = new String[]` compiles but risks `ArrayStoreException`     |
| Varargs                | Sugar (`int... nums`); only one, and it must be the last parameter     |
| `Arrays.asList(int[])` | Yields `List<int[]>` of size 1, not `List<Integer>`; box first         |
| Jagged arrays          | Arrays of arrays; rows may have different lengths                      |
