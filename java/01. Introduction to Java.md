## What is Java?

A general-purpose, **object-oriented programming language** plus a platform (runtime + standard libraries).

- Created by James Gosling at Sun Microsystems, 1995
- Now owned by Oracle Corporation (acquired Sun in 2010)
- Design for write once run anywhere, memory safety, automatic garbage collection, built-in concurrency
- Used for enterprise backends, Android, big data tools (Hadoop, Spark)

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

## Java Editions

Java has grown into multiple editions targeted at different environments:

### Java SE (Standard Edition)

The **core Java platform**: the language, JVM, and standard libraries (`java.lang`, `java.util`, `java.io`, `java.time`, `java.nio`, etc.).

- What most developers think of as "Java"
- Includes everything needed for desktop apps, command-line tools

### Jakarta EE (Enterprise Edition)

Enterprise add-ons **on top of Java SE**: Servlets, JSP, JPA, EJB, JMS, etc. Used for server-side / web / distributed systems.

- **Formerly Java EE**: renamed to Jakarta EE in **2018** when Oracle transferred it to the **Eclipse Foundation**: `javax.*` packages renamed to `jakarta.*` starting Jakarta EE 9

### Java ME (Micro Edition)

Legacy edition for **mobile / embedded / feature-phones**. Effectively **dead** for modern development.

- Modern mobile development moved to **Android** (uses Java + Kotlin but on a completely separate platform — different runtime, different APIs)
- Android is **NOT** a Java Edition — it just happens to use the Java language

## Java Features

Java's original whitepaper listed ~11 buzzwords. The five that matters in practice:

### Object-Oriented

Everything (except primitives) is an object. Code is organized around classes, inheritance, polymorphism, encapsulation.

```java
class Employee {
    private String name;
    public Employee(String name) { this.name = name; }
    public String getName() { return name; }
}
```

### Platform Independent

Compile once → run on any OS with a JVM. The compiler produces **bytecode** (`.class`), not machine code. Each platform's JVM interprets / JIT-compiles that bytecode to native machine code.

- Same `.class` file runs on Windows, macOS, Linux
- Developers say "compile once, run anywhere" — accurate at the bytecode level

### Secured

- **No pointers**: can't manipulate memory directly, preventing buffer overflow classes of bugs
- **Automatic memory management**: garbage collector; no manual `free()` / dangling pointers
- **Bytecode verification**: JVM checks class files before execution (prevents malformed bytecode exploits)

### Multithreaded

First-class concurrency and multi threading support.

### Compiled & Interpreted

Not pure-compiled (like C++) and not pure-interpreted (like classic Python). **Both**:

1. **`javac`** (compiler) → `.java` source becomes `.class` bytecode
2. **JVM** (runtime) → interprets bytecode line by line AT FIRST, then **JIT-compiles hot code paths** to native machine code for speed

This hybrid model is why Java has "warm-up" time — the first few requests are interpreted, later ones are near-native speed.

## Q&A

1. **What is the technical relationship between Java and JavaScript?**
   There is no technical relationship. JavaScript was originally named "LiveScript," and Netscape renamed it to "JavaScript" in 1995 purely to capitalize on the massive industry hype surrounding Java at the time. They are different languages with different syntax, ecosystems, and runtime environments.

2. **How do Jakarta EE and Spring compare in the modern Java ecosystem?**
   Jakarta EE (formerly Java EE) is a set of specifications (standardized APIs like JPA, Servlets, and CDI). Spring is a framework that provides its own implementations and abstractions. While Spring often uses Jakarta EE specifications (like JPA), most modern enterprise Java applications prefer Spring Boot for its ease of use and opinionated defaults.

3. **Is Android considered a "Java Edition"?**
   No. While Android uses the Java programming language (and Kotlin), it is a completely separate platform. It does not use a standard Java Virtual Machine (JVM); instead, it uses the Android Runtime (ART). It also has its own standard library and does not support many core Java SE APIs like AWT, Swing, or JavaFX.

4. **Why is there confusion between Java version numbers like "1.8" and "Java 8"?**
   Historically, Java versions followed a `1.x` naming convention (e.g., `1.1`, `1.2`). Starting with Java 5, Oracle shifted the public naming to simple integers, but the internal versioning remained `1.x` for legacy compatibility. This is why running `java -version` on a Java 8 installation may still report `1.8.xxx`. From Java 10 onwards, the internal and public versions were finally aligned.
