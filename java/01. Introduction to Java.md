A general-purpose **object-oriented programming language** plus a platform (runtime + standard libraries).

- Created by James Gosling at Sun Microsystems, 1995
- Now owned by Oracle Corporation (acquired Sun in 2010)
- Designed for write-once-run-anywhere, memory safety, automatic garbage collection, and built-in concurrency
- Used for enterprise backends, Android, and big-data tools (Hadoop, Spark)

    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

## Java Editions

Java has grown into multiple editions targeted at different environments:

### Java SE (Standard Edition)

The **core Java platform**: the language, JVM, and standard libraries (`java.lang`, `java.util`, `java.io`, `java.time`, `java.nio`, etc.).

- What most developers think of as "Java"
- Includes everything needed for desktop apps and command-line tools

### Jakarta EE (Enterprise Edition)

Enterprise add-ons on top of Java SE: Servlets, JSP, JPA, EJB, JMS, etc. Used for server-side, web, and distributed systems.

- Formerly Java EE: renamed to Jakarta EE in 2018 when Oracle transferred it to the Eclipse Foundation. The `javax.*` packages became `jakarta.*` starting with Jakarta EE 9.

### Java ME (Micro Edition)

Legacy edition for mobile, embedded, and feature-phones; effectively dead for modern development.

- Modern mobile development moved to Android, which uses Java and Kotlin but on a separate platform (different runtime, different APIs)
- Android is not a Java Edition; it just uses the Java language

## Java Features

Java's original whitepaper listed ~11 buzzwords; the five that matter in practice:

- **Object-Oriented**: everything except primitives is an object; code is organized around classes, inheritance, polymorphism, and encapsulation.
- **Platform Independent**: `javac` compiles source to **bytecode** (`.class`), not machine code, so the same `.class` file runs on any OS with a JVM (Windows, macOS, Linux). Each platform's JVM interprets or JIT-compiles that bytecode to native code.
- **Secured**: no pointers (no direct memory access, preventing buffer-overflow bugs); automatic memory management via the garbage collector (no manual `free()` or dangling pointers); bytecode verification (the JVM validates class files before execution).
- **Multithreaded**: first-class concurrency and multithreading support.
- **Compiled & Interpreted**: both. `javac` compiles `.java` to `.class` bytecode, then the JVM interprets it at first and JIT-compiles hot paths to native code. This hybrid model is why Java has "warm-up" time: early calls are interpreted, later ones run near-native.

## Q&A

1. **How do Jakarta EE and Spring compare in the modern Java ecosystem?**
Jakarta EE (formerly Java EE) is a set of specifications (standardized APIs like JPA, Servlets, and CDI). Spring is a framework that provides its own implementations and abstractions. While Spring often uses Jakarta EE specifications (like JPA), most modern enterprise applications prefer Spring Boot for its ease of use and opinionated defaults.

2. **Is Android considered a "Java Edition"?**
No. While Android uses the Java programming language (and Kotlin), it is a completely separate platform. It does not use a standard JVM; instead it uses the Android Runtime (ART). It also has its own standard library and does not support many core Java SE APIs like AWT, Swing, or JavaFX.

4. **Why is there confusion between Java version numbers like "1.8" and "Java 8"?**
Historically, Java versions followed a `1.x` convention (e.g., `1.1`, `1.2`). Starting with Java 5, Oracle shifted the public naming to simple integers, but the internal versioning remained `1.x` for legacy compatibility. This is why `java -version` on a Java 8 installation may still report `1.8.xxx`. From Java 10 onward, the internal and public versions were finally aligned.

## Quick Revision Cheat Sheet

| Concept                | Remember                                                       |
| ---------------------- | -------------------------------------------------------------- |
| Java SE                | Core platform (language, JVM, standard libs)                   |
| Jakarta EE             | Enterprise (server-side APIs); formerly Java EE                |
| Java ME                | Mobile / embedded (legacy); superseded by Android              |
| Android                | Not a Java Edition, separate platform using the Java language  |
| Bytecode               | `.class`, compiled by `javac`, runs on any JVM                 |
| Platform Independent   | Same `.class` runs anywhere; JIT compiles to native at runtime |
| Compiled + Interpreted | `javac` compiles to bytecode; JVM interprets + JIT-compiles    |
| Security               | No pointers, GC, bytecode verification                         |
