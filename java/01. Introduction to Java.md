Java is a general-purpose, object-oriented programming language and a platform (a runtime plus standard libraries).

- Created by James Gosling at Sun Microsystems, 1995; now owned by Oracle (which acquired Sun in 2010).
- Designed for write-once-run-anywhere portability, memory safety, automatic garbage collection, and built-in concurrency.
- Used for enterprise backends, Android, and big-data tools (Hadoop, Spark).
- "Java" is both a language and a platform: the JVM also hosts other languages such as Kotlin, Scala, and Groovy.

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

## Java Editions

Java targets different environments through several editions.

### Java SE (Standard Edition)

The core platform: the language, the JVM, and the standard libraries (`java.lang`, `java.util`, `java.io`, `java.time`, `java.nio`). This is what most developers mean by "Java" and covers desktop apps and command-line tools.

### Jakarta EE (Enterprise Edition)

Enterprise APIs layered on top of Java SE: Servlets, JSP, JPA, EJB, JMS, and more, for server-side, web, and distributed systems. Formerly Java EE, it was renamed when Oracle transferred it to the Eclipse Foundation in 2018; the `javax.*` packages became `jakarta.*` starting with Jakarta EE 9.

### Java ME (Micro Edition)

Legacy edition for mobile, embedded, and feature phones, effectively dead for modern work. Modern mobile development moved to Android, which uses the Java language (and Kotlin) but is a separate platform with its own runtime and APIs, not a Java edition.

## Java Features

The original whitepaper listed about eleven buzzwords; these five matter in practice.

- **Object-Oriented**: everything except primitives is an object; code is organized around classes, inheritance, polymorphism, and encapsulation.
- **Platform Independent**: `javac` compiles source to bytecode (`.class`), not machine code, so one `.class` file runs on any OS that has a JVM (Windows, macOS, Linux).
- **Secured**: no pointers (no direct memory access, avoiding buffer overflows), automatic memory management via the garbage collector (no manual `free()` or dangling pointers), and bytecode verification before execution.
- **Multithreaded**: first-class concurrency support in the language and standard library.
- **Compiled & Interpreted**: a hybrid model. The JVM interprets bytecode at first and JIT-compiles hot paths to native code, which is why Java apps "warm up", early calls are interpreted and later ones run near-native.

## Q&A

1. **How do Jakarta EE and Spring compare in the modern ecosystem?**
Jakarta EE is a set of standardized specifications (JPA, Servlets, CDI); Spring is a framework with its own implementations and abstractions that often builds on those specs (for example JPA). Most new enterprise apps favour Spring Boot for its opinionated defaults and ease of use.

2. **Is Android a "Java Edition"?**
No. Android uses the Java language (and Kotlin) but is a separate platform: it runs on the Android Runtime (ART), not a standard JVM, and ships its own libraries without many Java SE APIs such as AWT, Swing, or JavaFX.

3. **Why the confusion between version numbers like "1.8" and "Java 8"?**
Early Java used a `1.x` scheme (`1.1`, `1.2`); the public name switched to plain integers around Java 5, but the internal version stayed `1.x`, so `java -version` on Java 8 still reports `1.8.x`. Public and internal numbers were aligned from Java 9/10 onward, after which Java moved to a six-month release cadence with LTS versions (8, 11, 17, 21) that receive long-term support.

## Quick Revision Cheat Sheet

| Concept                | Remember                                                       |
| ---------------------- | -------------------------------------------------------------- |
| Java SE                | Core platform: language, JVM, standard libs                    |
| Jakarta EE             | Enterprise server-side APIs; formerly Java EE (`javax`→`jakarta`) |
| Java ME                | Mobile / embedded (legacy); superseded by Android              |
| Android                | Not a Java edition; separate platform (ART) using the Java language |
| Bytecode               | `.class` compiled by `javac`, runs on any JVM                  |
| Platform Independent   | Same `.class` runs anywhere; JVM JIT-compiles to native        |
| Compiled + Interpreted | `javac` → bytecode; JVM interprets, then JIT-compiles hot paths |
| Security               | No pointers, GC, bytecode verification                         |
| Releases               | Six-month cadence since Java 9; LTS = 8, 11, 17, 21            |
