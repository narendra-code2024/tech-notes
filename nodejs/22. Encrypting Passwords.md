## Why Encrypt Passwords?

- Never store passwords as plain text in the database
- Use **hashing**: a one-way function that converts passwords into an irreversible string
- Even if the database is compromised, the actual passwords remain safe

---

## bcrypt Package

```bash
npm install bcrypt
```

- Industry standard for password hashing in Node.js
- Uses **salt rounds** to add randomness and make each hash unique

### Hashing a Password

```js
const bcrypt = require('bcrypt');

const hashPassword = async (plainPassword) => {
    const saltRounds = 10;
    const hashedPassword = await bcrypt.hash(plainPassword, saltRounds);
    return hashedPassword;
};
```

### Example Output

```
Plain:  "MySecret@123"
Hashed: "$2b$10$N9qo8uLOickgx2ZMRZoMye3Z6tY/3cWMcUi7kg/IcHmNsy38GFTKS"
```

- Each call to `bcrypt.hash` produces a **different hash** (because of the random salt)
- But `bcrypt.compare` can still match the plain password against any of those hashes

### Comparing Passwords (Login)

```js
const isMatch = await bcrypt.compare(plainPassword, hashedPassword);
// Returns true if match, false otherwise
```

```
bcrypt.compare("MySecret@123", "$2b$10$N9qo8uLO...")  -->  true
bcrypt.compare("WrongPass",    "$2b$10$N9qo8uLO...")  -->  false
```

- `bcrypt.hash` -- hash a password before saving to DB
- `bcrypt.compare` -- compare plain text password with stored hash (for login)
