## First Node.js Program (using REPL)

- **REPL** stands for **Read-Eval-Print Loop**: The quickest and easiest way to write and run JavaScript code in Node.js
- To enter Node REPL: type `node` in the terminal and press Enter
- Node REPL lets you run JavaScript interactively, similar to the browser console
- You are successfully running code in the Node.js runtime environment
- That's why Node.js is called a **JavaScript runtime environment**

```bash
$ node
> var a = 10;
> var b = 20;
> a + b
30
> console.log("Hello from Node REPL");
Hello from Node REPL
```

## First JavaScript File in Node.js

- You can name the file anything like `app.js`, `index.js`, `main.js`, `script.js`
- Run it using: `node filename.js`
- This is how you execute a JavaScript file inside the JS runtime environment, which is Node.js

```js
// app.js
var a = 10;
var b = 20;
console.log(a + b);  // 30
```

```bash
$ node app.js
30
```

## Global Object in Node.js

|Environment|Global Object|`this` in Global Scope|
|---|---|---|
|**Browser**|`window`|Points to `window`|
|**Web Workers**|`self`|Points to `self`|
|**Node.js**|`global`|Returns empty object `{}`|
|**All (standard)**|`globalThis`|Universal standard|

### Browser

```js
// In the browser
console.log(window);    // Window {…}
console.log(this);      // Window {…}  (points to window)
console.log(self);      // Window {…}
console.log(frames);    // Window {…}
console.log(this === window);  // true

// window, this, self, frames all refer to the same global object
```

- The `window` object is provided by the **browser**, not by the V8 engine

```js
// In the browser
console.log(global);  // ReferenceError: global is not defined
// "global" does NOT exist in the browser -- it's a Node.js thing
```

### Node.js

```js
// In Node.js
console.log(global);    // Object [global] { setTimeout, setInterval, ... }
console.log(this);      // {}  (empty object!)
console.log(this === global);  // false

// Functions like setTimeout, setInterval, setImmediate come from global object
```

- In Node.js, the global object is called `global` and it's provided by **Node.js**: `this` in global scope returns **empty object `{}`**, not the `global` object
- If you log `this` in the global scope, it returns an empty object `{}`, not the global object

### Web Workers

```js
// In Web Workers (browser)
console.log(self);       // DedicatedWorkerGlobalScope
// There is a concept of web workers (aim is: these they started
// reinventing global object), with a
// new context created using a web worker
```

## globalThis (Standardization)

- In **2020**, the **OpenJS Foundation** proposed a standard global object across all runtime environments
- Some suggested using `global` as the standard, but concerns arose about potential **naming conflicts** (libraries, browsers, or other environments may already use `window`, `this`, `self`, `frames`, or `global` as the standard global object)
- Solution: **`globalThis`**: a new standard keyword that refers to the global object across **all** JavaScript runtime environments
- In Node.js, `globalThis` is **not a copy** of `global` -- it points to the **same reference**

```js
// Browser
console.log(globalThis === window);  // true

// Node.js
console.log(globalThis === global);  // true

// Web Worker
console.log(globalThis === self);    // true
```
