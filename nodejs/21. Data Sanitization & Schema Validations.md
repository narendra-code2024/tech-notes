## Two Levels of Validation

|Level|Where|Purpose|
|---|---|---|
|**Schema-Level**|Model file (Mongoose)|Define constraints on fields (required, type, min/max, enum, etc.)|
|**API-Level**|Route handler / middleware|Validate request data before it reaches the DB|

---

## Schema-Level Validation (Mongoose)

```js
const userSchema = new mongoose.Schema({
    firstName: {
        type: String,
        required: [true, "First name is required"],
        minLength: [2, "Name must be at least 2 characters"]
    },
    email: {
        type: String,
        required: true,
        unique: true,
        lowercase: true,
        trim: true
    },
    age: {
        type: Number,
        min: 18,
        max: 100
    },
    gender: {
        type: String,
        enum: ["male", "female", "other"]
    }
});
```

---

## API-Level Validation (validator package)

```bash
npm install validator
```

- Use the `validator` package to validate email, strong password, URLs, and many more

```js
const validator = require('validator');

// In your route handler or middleware
if (!validator.isEmail(req.body.email)) {
    return res.status(400).send("Invalid email");
}

if (!validator.isStrongPassword(req.body.password)) {
    return res.status(400).send("Weak password");
}
```
