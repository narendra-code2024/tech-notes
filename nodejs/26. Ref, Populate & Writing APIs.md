## MongoDB ref & populate

- `ref` creates a **reference** (relationship) between two collections
- `populate()` replaces the ObjectId with the **actual document** from the referenced collection

### Defining a Reference

```js
const connectionRequestSchema = new mongoose.Schema({
    fromUserId: {
        type: mongoose.Schema.Types.ObjectId,
        ref: 'User',       // references the User collection
        required: true
    },
    toUserId: {
        type: mongoose.Schema.Types.ObjectId,
        ref: 'User',
        required: true
    },
    status: {
        type: String,
        enum: ["interested", "ignored", "accepted", "rejected"],
        required: true
    }
});
```

### Using populate()

```js
// Without populate -- returns ObjectIds
const requests = await ConnectionRequest.find({ toUserId: userId });
// { fromUserId: "665a...", toUserId: "665b...", status: "interested" }

// With populate -- returns full user documents
const requests = await ConnectionRequest.find({ toUserId: userId })
    .populate('fromUserId', 'firstName lastName email');
// { fromUserId: { firstName: "Narendra", lastName: "Katta", email: "..." }, ... }
```

- Second argument to `populate()` selects **specific fields** to include

## MongoDB Query Operators

Useful operators for building complex queries:

|Operator|Purpose|Example|
|---|---|---|
|`$eq`|Equal|`{ status: { $eq: "accepted" } }`|
|`$ne`|Not equal|`{ status: { $ne: "rejected" } }`|
|`$in`|Matches any in array|`{ status: { $in: ["interested", "accepted"] } }`|
|`$nin`|Not in array|`{ status: { $nin: ["rejected", "ignored"] } }`|
|`$and`|All conditions must match|`{ $and: [{ status: "accepted" }, { fromUserId: id }] }`|
|`$or`|Any condition can match|`{ $or: [{ fromUserId: id }, { toUserId: id }] }`|
