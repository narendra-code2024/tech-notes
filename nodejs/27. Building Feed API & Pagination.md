## Feed API

- The feed API returns a list of **users that the logged-in user has NOT yet connected with**: Should exclude:
  - The logged-in user themselves
  - Users already sent/received connection requests
  - Users already connected

---

## Pagination

- Loading all users at once is **not scalable**: use pagination
- MongoDB provides `skip()` and `limit()` for pagination

```js
// page 1: skip 0, limit 10
// page 2: skip 10, limit 10
// page 3: skip 20, limit 10

const page = parseInt(req.query.page) || 1;
const limit = parseInt(req.query.limit) || 10;
const skip = (page - 1) * limit;

const users = await User.find({ _id: { $nin: excludedUserIds } })
    .skip(skip)
    .limit(limit)
    .select('firstName lastName email age');
```

### How skip & limit Work

|Parameter|Purpose|
|---|---|
|`skip(n)`|Skip the first `n` documents|
|`limit(n)`|Return at most `n` documents|
|`select('field1 field2')`|Return only specified fields|

```
Total Users: 50

Page 1: skip(0).limit(10)   --> Users 1-10
Page 2: skip(10).limit(10)  --> Users 11-20
Page 3: skip(20).limit(10)  --> Users 21-30
```
