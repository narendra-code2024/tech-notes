## Initialize Your Project

```bash
mkdir my-project && cd my-project
npm init
```

- Creates a `package.json` file -- the **JSON configuration file** for your project
- Contains project metadata like name, version, dependencies, etc.
- Create a `src` folder (common convention for source code across the industry)
- Inside `src`, create `app.js` -- the **entry point** of the application

---

## Install Express

```bash
npm install express
```

- Adds `node_modules` folder -- contains all source code of express
- Adds `express` under **dependencies** in `package.json`
- Creates `package-lock.json` -- **locks exact versions** of installed packages

---

## Semantic Versioning

Version format: **major.minor.patch** (e.g., `4.19.2`)

|Change|Type|Example|
|---|---|---|
|**Patch**|Small changes or bug fixes|`4.19.2` --> `4.19.3`|
|**Minor**|New features without breaking changes|`4.19.3` --> `4.20.0`|
|**Major**|Breaking changes|`4.20.3` --> `5.0.0`|

```
^4.19.2  --> allows updates within 4.x.x, but NOT 5.x.x
~4.19.2  --> allows updates within 4.19.x, but NOT 4.20.x
```

- Even with `^` in `package.json`, the exact version used is locked in `package-lock.json`
- `.bin` folder inside `node_modules` contains CLI binaries for packages

## Create Server & Listen

```js
const express = require('express');
const app = express();

app.use('/', (req, res) => {
    res.send('Hello World!');
});

app.listen(3000, () => {
    console.log('Server running on port 3000');
});

// Output: Server running on port 3000
// Visit http://localhost:3000 --> "Hello World!"
```

- Express server is a web server that **listens on a port**: Anyone can send a request to it and get a response
- In the above example, whatever the request is, it responds with "Hello World!"

---

## Handling Different Routes

```js
app.use('/hello', (req, res) => {
    res.send('Hello!');
});

app.use('/dashboard', (req, res) => {
    res.send('Dashboard');
});

// GET /hello     --> "Hello!"
// GET /dashboard --> "Dashboard"
```

- Use `app.use()` with the **route path** as the first argument
- Problem: if `/` route is defined first, it catches all requests (since every route starts with `/`)

## nodemon -- Auto-Reload on Changes

- Normally, you need to rerun `node src/app.js` after every change
- **nodemon** auto-restarts the server on file changes

```bash
npm i -g nodemon        # install globally
nodemon src/app.js       # use instead of node
```

- `-g` installs globally so it can be used in any project
- Add this command under `scripts` in `package.json` for easier use:

```json
// package.json
{
  "scripts": {
    "start": "node src/app.js",
    "dev": "nodemon src/app.js"
  }
}
```

```bash
npm run dev    # starts server with auto-reload
npm start      # starts server without auto-reload (for production)
```
