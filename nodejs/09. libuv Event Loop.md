## Event Loop

```
                  EVENT LOOP

            +-------+
            | Timer | (setTimeout, setInterval)
            +---+---+
                |
                v
  +-------+   +-------+
  | Close | <-- | Poll  | (I/O callbacks, fs, net, http, crypto)
  +---+---+   +---+---+
      ^            |
      |            v
      +---  +-------+
            | Check | (setImmediate)
            +-------+
            
  Close --next tick--> Timer (loop continues)
```

---

## Event Loop Phases

The Event Loop has **four major phases**:

| Phase | What Executes | Examples |
| --- | --- | --- |
| **1. Timer** | `setTimeout` and `setInterval` callbacks | Delayed execution |
| **2. Poll** | I/O callbacks (most important phase) | File reading, API/Network requests, DB queries, crypto |
| **3. Check** | `setImmediate` callbacks | Run after poll phase |
| **4. Close** | Cleanup operations | `socket.on("close")` |

- Timers execute on or after a **specified delay** (not exact) -- the OS scheduling or other callbacks can delay them

---

## Microtask Queue (Priority Queue)

**Before each phase** in the Event Loop, Node.js first executes microtasks (higher-priority):

| Priority | Microtask |
| --- | --- |
| **1 (Highest)** | `process.nextTick` callbacks |
| **2** | Promise callbacks (`.then`, `.catch`, `.finally`) |

- Microtask queue is **not a phase**: it runs **before entering each phase**: Even if you're in the middle of moving from Timer to Poll, microtasks run first

---

## How the Event Loop Works

1. All callbacks wait in their respective **callback queues** (Timers, I/O, Check, Close)
2. The Event Loop **continuously checks** these queues
3. When the **call stack in V8 is empty**, it picks the next callback from the queue and pushes it to the stack for execution

```
        +-------+      +------+      +-------+      +-------+
   +--> | Timer | ---> | Poll | ---> | Check | ---> | Close | --+
   |    +-------+      +------+      +-------+      +-------+   |
   |                                                             |
   +-------------------- next tick of loop ---------------------+

   Before EACH phase:  nextTick queue --> Promise queue --> Phase
```

---

## Event Loop and the Poll Phase

- If there are **no pending callbacks** in any queue, the Event Loop **waits in the Poll phase** for ongoing I/O operations to complete
- Once an I/O operation finishes, its callback is pushed to the event loop for execution

---

## Node.js vs Browser Event Loop

| Node.js Event Loop | Browser Event Loop |
| --- | --- |
| **Waits** in Poll phase when no work | Does **not** wait -- continues rendering |
| Designed for server-side **I/O-heavy** tasks | Optimized for **UI rendering** and responsiveness |
| Uses libuv | Uses browser engine's event loop |
| Has `setImmediate`, `process.nextTick` | Has `requestAnimationFrame` |

- In Node.js, the Timer phase is designed for **setTimeout/setInterval**, while `setImmediate` runs in Check phase (optimized for I/O callbacks context)

## Event Loop Example 1

```js
const fs = require("fs");

setImmediate(() => console.log("setImmediate"));

setTimeout(() => console.log("setTimeout"), 0);

Promise.resolve().then(() => console.log("Promise"));

process.nextTick(() => console.log("nextTick"));

fs.readFile("./file.txt", "utf-8", () => {
    console.log("File I/O");
});

console.log("Sync code");
```

```
Output:
Sync code            <-- call stack (runs first)
nextTick             <-- microtask (highest priority)
Promise              <-- microtask (after nextTick)
setTimeout           <-- timer phase
setImmediate         <-- check phase
File I/O             <-- poll phase (depends on file read time)
```

---

## Event Loop Example 2

```js
fs.readFile("./file.txt", "utf-8", () => {
    console.log("File I/O callback");

    setTimeout(() => console.log("setTimeout inside I/O"), 0);

    setImmediate(() => console.log("setImmediate inside I/O"));

    process.nextTick(() => console.log("nextTick inside I/O"));
});
```

```
Output:
File I/O callback
nextTick inside I/O       <-- microtask runs first
setImmediate inside I/O   <-- check phase (always before timer when inside I/O)
setTimeout inside I/O     <-- timer phase
```

- **Inside an I/O callback**, `setImmediate` always runs **before** `setTimeout(cb, 0)` because after Poll phase, Check phase comes next

---

## process.nextTick Recursion

- If a `process.nextTick()` callback schedules **another** `process.nextTick()` recursively, all nextTick callbacks execute **in order** before the Event Loop moves to the next phase
- The Event Loop proceeds **only when the nextTick queue is empty**: This can **starve** other phases if used carelessly

---

## Difference between Promise .then and process.nextTick

- Both are **microtasks** but `process.nextTick` has **higher priority**: `process.nextTick` queue is drained completely before Promise queue
- Be careful to know the difference (this is part of Namaste Node.js)
