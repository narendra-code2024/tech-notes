## JWT (JSON Web Token)

- Every JWT token has **three parts**:

```
Header.Payload.Signature

eyJhbGciOiJIUzI1NiJ9 . eyJ1c2VySWQiOiIxMjMifQ . abc123signature
          |                       |                      |
       Header                 Payload               Signature
    (algorithm)            (user data)          (verification key)
```

---

## JWT Operations

### Sign (Create Token)

```js
const jwt = require('jsonwebtoken');

// Create a token
const token = jwt.sign({ userId: "123" }, "your-secret-key", { expiresIn: "1d" });
```

### Verify (Validate Token)

```js
// Verify and decode a token
const decoded = jwt.verify(token, "your-secret-key");
console.log(decoded.userId);  // "123"
```

## cookie-parser

- Developed by the Express.js team
- A **middleware** that enables you to read cookies from incoming requests

```bash
npm install cookie-parser
```

```js
const cookieParser = require('cookie-parser');
app.use(cookieParser());

// Setting a cookie
res.cookie('token', jwtToken, { httpOnly: true, expires: new Date(Date.now() + 86400000) });

// Reading a cookie
const { token } = req.cookies;
```

## JWT in Mongoose Schema Methods

- We can do `jwt.sign` in the signup API, but **good practice** is to keep it in the schema
- Since each user will have a unique JWT token, offload this to a **schema method**

```js
const userSchema = new mongoose.Schema({ /* fields */ });

// IMPORTANT: Use normal function, NOT arrow function
userSchema.methods.getJWT = function () {
    const token = jwt.sign({ _id: this._id }, "your-secret-key", { expiresIn: "1d" });
    return token;
};
```

### Why Normal Function Instead of Arrow Function?

- `this` refers to the **current user instance** (the document)
- Arrow functions don't have their own `this` -- they inherit from the enclosing scope
- Normal functions bind `this` to the calling object (the user instance)

```js
// In controller (app.js)
const user = await User.findOne({ email });
const token = user.getJWT();   // this = user instance
res.cookie('token', token);
```

This makes controller logic **clean**: JWT creation logic stays in the schema.
