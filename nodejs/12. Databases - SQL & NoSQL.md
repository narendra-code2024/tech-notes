## What is a Database?

- An **organized collection of data** stored in a structured format for easy access, management, and updating

---

## What is DBMS?

- **DBMS** (Database Management System) is a software layer on top of a database
- Provides tools and interfaces to manage data, perform CRUD operations, ensure data integrity and security

---

## Types of Databases

### Relational Databases (RDBMS) -- SQL

- **SQL** = Structured Query Language -- used to interact with relational databases
- Examples: **MySQL, PostgreSQL**: Data organized in **tables** (structured schema)
  - **Rows** = individual records
  - **Columns** = fields

### NoSQL Databases ("Not Only SQL")

- Non-tabular format, **schema-less** (or dynamic schema)
- Types include:
  - **Document** (e.g., MongoDB)
  - **Key-Value** (e.g., Redis)
  - **Wide Column** (e.g., Cassandra)
  - **Graph** (e.g., Neo4j)

---

## RDBMS vs MongoDB

|RDBMS|MongoDB (Document DB)|
|---|---|
|Tables|Collections|
|Rows / Records|Documents|
|Columns / Fields|Fields (keys)|
|Fixed Schema|Dynamic / Flexible Schema|

A row in MySQL becomes a **document** in MongoDB, and a column becomes a **key** in that document.

### Example: SQL Row vs MongoDB Document

```js
// SQL Row (table: users)
// | id | name     | age |
// | 1  | "Akshay" | 25  |

// MongoDB Document (collection: users)
{
    _id: ObjectId("..."),
    name: "Akshay",
    age: 25
}
```

---

## MongoDB

- **Document-oriented** database -- stores data in **JSON-like objects**: Works well with JSON and JavaScript Objects -- **very compatible** with Node.js
- Documents stored in **collections** (similar to tables)

### Why MongoDB?

- **Distributed computing**: built for horizontal scaling
- Supports **sharding** (distributing data across servers)
- Enables high availability and fault tolerance
- Ideal for applications with large-scale data and global access

### MongoDB Atlas

- **Cloud-hosted** MongoDB service (managed by MongoDB)
- Offers a **free tier** for development and testing
- Easy to set up and scale

---

## Mongoose (ODM)

- **Object Data Modeling** library for MongoDB in Node.js
- Simplifies:
  - Defining schemas
  - Querying the database
  - Validation and middleware

```
Raw HTTP Server  -->  Express.js (easier server logic)
Raw MongoDB Driver --> Mongoose (easier DB operations)
```

Just like using Express instead of raw `http` for server logic, we use Mongoose instead of the raw `mongodb` driver for DB operations.
