## Route Matching Order in Express

- Express processes routes **top to bottom**, in the order they are defined
- `app.use('/')` matches **every** request (since every route starts with `/`)
- `app.use('/hello')` matches `/hello`, `/hello/2`, `/hello/anything` (prefix-based matching)

### Problem: Catch-All Route First

```js
// BAD -- / catches everything, /hello and /dashboard never reached
app.use('/', (req, res) => { res.send('Welcome!'); });
app.use('/hello', (req, res) => { res.send('Hello!'); });
app.use('/dashboard', (req, res) => { res.send('Dashboard'); });
```

### Fix: Specific Routes Before Catch-All

```js
// GOOD -- specific routes first, catch-all last
app.use('/hello', (req, res) => { res.send('Hello!'); });
app.use('/dashboard', (req, res) => { res.send('Dashboard'); });
app.use('/', (req, res) => { res.send('Welcome!'); });
```

### Prefix Matching Example

```js
app.use('/hello/2', (req, res) => { res.send('Hello World2!'); });
app.use('/hello', (req, res) => { res.send('Hello World!'); });

// /hello/2  --> "Hello World2!"
// /hello    --> "Hello World!"
// /hello/x  --> "Hello World!" (prefix match on /hello)

// If reversed: /hello defined first catches /hello/2 too!
```

## app.use() vs app.get() / app.post()

|Method|Matches HTTP Methods|
|---|---|
|`app.use('/path')`|**All** methods (GET, POST, PUT, DELETE, etc.)|
|`app.get('/path')`|Only **GET**|
|`app.post('/path')`|Only **POST**|
|`app.put('/path')`|Only **PUT**|
|`app.delete('/path')`|Only **DELETE**|

- When hitting a URL in the browser, it's a **GET request** by default
- You can't directly access POST/PUT/DELETE APIs in the browser -- use **Postman**, `curl`, or `fetch`/`axios`

## Route Patterns & Regex

|Pattern|Matches|
|---|---|
|`/ab?c`|`b` is optional -- `/abc` and `/ac`|
|`/ab+c`|One or more `b` -- `/abc`, `/abbc`, `/abbbc`|
|`/ab*cd`|Anything between `ab` and `cd` -- `/abcd`, `/abXYZcd`|
|`/a(bc)?d`|`bc` is optional group -- `/ad` and `/abcd`|
|`/a(bc)+d`|One or more `bc` -- `/abcd`, `/abcbcd`|
|`/a/`|Regex -- any route containing `a`|

## Query Parameters (Optional Data)

- Passed in the URL after `?`

```
/user?userId=10&userName=narendra
```

```js
app.get('/user', (req, res) => {
    console.log(req.query.userId);      // "10"
    console.log(req.query.userName);    // "narendra"
});
```

---

## Route Parameters (Required / Dynamic Data)

- Passed directly in the route path

```
/user/123/naren
```

```js
app.get('/user/:id/:name', (req, res) => {
    console.log(req.params.id);    // "123"
    console.log(req.params.name);  // "naren"
});
```
