## Why Are Module Variables Private?

- All the code of a module is **wrapped inside a function (an IIFE)**: so variables and functions are not accessible outside the module
- This ensures **encapsulation** and prevents name conflicts across different modules

---

## IIFE (Immediately Invoked Function Expression)

- An IIFE is a JavaScript function that is **executed immediately** after it is defined

```js
// IIFE syntax
(function () {
    console.log("I run immediately!");
})();
```

- Wrapping the function in parentheses turns it into a **function expression**, allowing it to invoke immediately

### Purpose of IIFE (in general and in Node.js)

1. **Immediate Execution**: executes code right away
2. **Privacy**: keeps variables and functions private to the scope
3. **Avoid Conflicts**: prevents variable/function name clashes in the global scope

---

## Functions Inside Functions

- It is valid in JavaScript to define a function within another function
- Inner functions have access to the **outer function's scope** but are **not accessible** from outside

```js
function outer() {
    const secret = "hidden";

    function inner() {
        console.log(secret);  // can access outer scope
    }

    inner();
}

outer();
// inner();  // ReferenceError -- not accessible outside outer()
```

---

## What Happens When You Use require()?

- Node.js wraps the entire module code inside a **special IIFE wrapper function**: This wrapper provides module-level scope and simulates private variables

```js
// Node.js module wrapper (what Node.js does behind the scenes)
(function (exports, require, module, __filename, __dirname) {
    // Your module code actually lives here
    const myVar = "private";
    module.exports = { /* exported items */ };
});
```

- Node.js passes `module`, `require`, `exports`, `__filename`, and `__dirname` to the IIFE

## How require(path) Works -- 5-Step Mechanism

```
require('./myModule')
        |
        v
  1. Resolving -----> Find the file path
        |
        v
  2. Loading -------> Read file from disk
        |
        v
  3. Wrapping ------> Wrap in IIFE
        |
        v
  4. Evaluating ----> Execute the code
        |
        v
  5. Caching -------> Store in cache
```

### Step 1: Resolving the Module

- Node.js determines the file path:
  - **Local file** (e.g., `./localPath`)
  - **JSON file** (e.g., `data.json`)
  - **Core Node module** (e.g., `fs`, `path`, `http`)
  - Or looks in `node_modules`

### Step 2: Loading the Module

- File content is read from disk
- Based on file type (`.js`, `.json`, or `.node`), Node chooses how to load and parse it

### Step 3: Wrapping the Module

- Code is wrapped inside an **IIFE** to create a private scope
- This gives each module its own private execution context

### Step 4: Evaluating the Module

- The wrapped function is executed
- The result of `module.exports` is returned from the module

### Step 5: Caching the Module

- Once loaded, Node **caches** the module
- Subsequent `require()` calls for the same module return the **cached result** instead of reloading
