## Why Express Router?

- As your app grows, defining all routes in a single `app.js` becomes **messy and hard to maintain**: `express.Router()` lets you create **modular, mountable route handlers**: Each router is like a mini Express app that handles a specific group of routes

---

## Using express.Router()

```js
// routes/auth.js
const express = require('express');
const authRouter = express.Router();

authRouter.post('/signup', (req, res) => {
    res.send("Signup");
});

authRouter.post('/login', (req, res) => {
    res.send("Login");
});

authRouter.post('/logout', (req, res) => {
    res.send("Logout");
});

module.exports = authRouter;
```

```js
// routes/profile.js
const express = require('express');
const profileRouter = express.Router();

profileRouter.get('/view', (req, res) => {
    res.send("View Profile");
});

profileRouter.patch('/edit', (req, res) => {
    res.send("Edit Profile");
});

module.exports = profileRouter;
```

```js
// app.js -- mount the routers
const authRouter = require('./routes/auth');
const profileRouter = require('./routes/profile');

app.use('/auth', authRouter);       // /auth/signup, /auth/login, /auth/logout
app.use('/profile', profileRouter); // /profile/view, /profile/edit
```

---

## Benefits

- **Separation of concerns**: each file handles one area of the API
- **Easier to maintain**: find and update routes quickly
- **Reusable**: mount same router on different paths if needed
- **Clean app.js**: entry point stays minimal and readable
