## Design

### HLD (High-Level Design)

- **Tech stack** selection
- **What microservices** are needed
- **What database** to use

### LLD (Low-Level Design)

- If your planning is good, coding is very easy -- otherwise you have to refactor again
- Detailed component-level design

- What **collections** are required?

---

## REST API Design

- Define all the APIs needed for the application
- Follow **RESTful conventions**: use appropriate HTTP methods with clear resource names

```
Good:
  GET    /profile        -- fetch profile
  POST   /profile        -- create profile
  PATCH  /profile        -- update profile
  DELETE /profile        -- delete profile

Bad:
  GET    /get-profile       -- redundant naming
  POST   /create-profile    -- method already implies action
```

- The **HTTP method + URL** together define the action -- avoid redundant naming like `/get-profile` or `/create-profile`
